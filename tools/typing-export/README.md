# タイプでスタディ データ取り出しツール（管理者専用）

タイピングゲームの分析データ（`sessions` / `players` / `rankings`）を、**管理者だけ**が手元に取り出すためのツールです。
Firebase の**サービスアカウント鍵**が必要で、これはプロジェクトのオーナー（あなた）しか発行できません。
＝**あなた以外は実行できません。**

> ⚠️ サービスアカウント鍵はセキュリティルールを無視して全データを読める強力な鍵です。
> **絶対に他人に渡さない・GitHubに上げない**でください（`.gitignore` で除外済み）。

---

## 1. サービスアカウント鍵を取得する

1. [Firebase コンソール](https://console.firebase.google.com) → プロジェクト **`type-study-88ced`**
2. 歯車 ⚙ → **プロジェクトの設定** → **「サービス アカウント」** タブ
3. **「新しい秘密鍵の生成」** → ダウンロードされる JSON ファイルを
   このフォルダに **`serviceAccount.json`** という名前で保存する

## 2. 準備（最初の1回だけ）

このフォルダで以下を実行（Node.js が必要）：

```
npm install
```

## 3. データを取り出す

```
npm run export
```

実行すると：

- **`out/` フォルダ**に CSV と JSON が書き出されます
  - `sessions` … 1プレイ1行（点数・正確率・単元・問題別の正誤など）
  - `sessions_flat` … 問題別配列を件数だけにした、表計算で開きやすい版
  - `players` … 端末ごとの累計データ
  - `rankings` … 全国ランキング
  - `difficult_questions` … 正答できなかった率が高い問題 上位30
- 画面に **日別アクティブ数 / 人気単元 / むずかしい問題** の集計が表示されます

`out/` フォルダも Git 管理外なので、データが誤って公開されることはありません。

---

## よくある質問

- **Q. 鍵を失くした／流出したかも** → コンソールの同じ画面で鍵を無効化＆再発行できます。すぐ無効化してください。
- **Q. `players` は read 禁止では？** → アプリ（ブラウザ）からは禁止ですが、このツールは管理者鍵なのでルールを越えて読めます。これが「管理者だけ」の仕組みです。
- **Q. 何回実行してもいい？** → はい。実行のたびに最新データで `out/` を上書きします。読み取りは無料枠内（件数が増えたら多少の読み取り回数を消費する程度）。
