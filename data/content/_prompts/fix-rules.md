# コンテンツ修正規則（理科クイズ・フラッシュカード）

ユーザーとのレビューで確立した修正基準。`data/content/**/*.json` の `flashcards` と `quiz.questions` に適用する。**`flashcards` と `quiz.questions` 以外のフィールドは変更しない。**

## 0. 中学生向け4択問題の作り方ポイント（早見・要点）

詳細は §B。まずこの7点を満たすこと（カッコ内は実レビューで直した例）。

1. **問題文に答えを書かない／透けさせない。** 答えの語・略称・定義的特徴を本文に入れない。
   - 例: 「電気には＋と−の何種類？」→ 答え「2種類」がバレる → 「電気には何種類？」。
   - 例: 「DNAは何文字で表される？」のように問いに答え（DNA）が入る循環問題は、DNAが答えになる形へ。
2. **答えが一意に定まる問いにする。** 「どうなる？」など複数解が出る問いは条件で限定するか削除。
   - 例: 「複製のとき遺伝子はどうなる？」→ 「ごくまれにどうなることで、親や祖先になかった形質が現れる？」＝「変化する」に一意化。
3. **4択は長さ・粒度・カテゴリ・形式をそろえる。** 正答だけ長い／具体的／複合（「AやB」）にしない＝それだけで答えがバレる。
   - 例: 正答「放射線や紫外線」だけ複合で長い → 「紫外線」に（他は水・二酸化炭素・酸素の単語）。
   - 例: 正答「核の中の染色体」→「染色体」（他が細胞質・細胞壁・葉緑体の単一構造名なので合わせる）。
   - 例: 選択肢の冗長な共通語尾「〜のはたらき」は削り、名詞で統一（問題文側で「何のはたらき」と受ける）。
4. **誤答（ディストラクタ）は同分野のもっともらしい誤りにする。** 別分野・ありえない・露骨に変な選択肢、「だけ/のみ」「すべて/絶対」等の手がかり語は禁止。正答位置はばらす。
5. **問題文・選択肢はコンパクトに。** 前置き・補足・カッコ書きは削り、補足は `explanation` へ。
   - 例: 「DNAという略称のもとになっている遺伝子の本体の正式な物質名は？」→「DNAの正式な物質名は？」。
6. **中学で一般的でない用語・範囲外は出さない。** 救えなければ削除。**削除しても id は振り直さない**（歯抜けのまま。番号参照を安定させる）。
   - 例: 「摩擦電気」カード/問題を削除。冗長・応用すぎカード（痕跡的器官の応用等）を削除。
7. **1問1論点・テキストだけで解ける・計算は検算。** 図/装置依存なら必要な数値・条件を本文に明記（例: 抵抗の材料に「ニクロム」を本文へ明記）。

## A. フラッシュカード（front=答え / back=問い）

### A-1. 削除する（修正で救えないもの）
- **手順・操作・説明文を答えにしている**：front が用語・値・現象でなく操作手順や説明文。
  例（削除済み）:「対物レンズとプレパラートを遠ざける」「観察物を視野の中央に置く」「枝の高さに合わせる」「原点を通る直線でむすぶ」「色だけでは決められない」。
- **答えが一意に定まらない**：記述に複数の答えが当てはまる。修正で一意化できなければ削除。
  例:「物体の質量をはかる器具は？」（電子てんびん/上皿てんびん）、「P波の速さは秒速何km？」（範囲6〜8km）、「代表的な火山岩は？」（流紋岩/安山岩/玄武岩）。
- **例を答えさせる**：back が「〜の例は？」「代表例は？」、または front が例の列挙・末尾「など」。
- **あいまいな基準を答えにする**：「体のつくり（で分類する）」のような漠然とした語句。
- **中学で扱わない応用・概念**：例「地層が逆転」を前提にした問題。

> 削除時は対象 id を取り除くだけで、**残りの id は振り直さない（歯抜けのまま）**。

### A-2. 修正する
- **問いと答えが噛み合っていない**：front が問いの言い換え、または「なぜ？/どうなって？」に対し理由・結果になっていない。
  例:「果実ができないのはなぜ？」→ front は「子房がないから」（×「果実ができない」）。「どうなって地震が起こる？」→「はね上がる」（×「はね上がるから」）。
- **問題文（back）内のカッコ書きは原則削除**（補足・別名・言い換え）。例:「地震の規模（放出されるエネルギー）を表すものは？」→「地震の規模を表すものは？」。情報は explanation へ。
- **「約／およそ」の整合**：前提が厳密な仮定（「100gの重力を1Nとする」「音の速さを340m/sとする」）なら答えに「約/およそ」は不要。前提自体に「約」があれば答えも「約」のまま。
- **中学で一般的でない表現を平易に**：例「地層が逆転していない場合」→「ふつうに積み重なった地層では」。
- **不自然な言い回しを直す**：
  - 「〜を答えると？」→「〜は？」「〜を3つあげると？」など自然に。
  - 他動詞/自動詞の誤り：「空気を…耳まで伝える」→「空気の中を…耳まで伝わる」。
  - 余分・紛らわしい情報を削る：高さ（ピッチ）を問うのに振幅の情報を混ぜない → 振動数に絞る。
- **難しい用語は explanation に軽く補足**（例:「同心円」は中心が同じ円が重なった形、と一言添える）。
- **答えの粒度をそろえる**：抽象表現より具体名をメインに（例: シジミの環境 → front「河口や湖」、抽象説明は explanation）。
- front の難読漢字にはルビ `{漢字|よみ}`（読みは指定があればそれに従う）。

## B. クイズ（4択）

### B-1. 選択肢の作り（最重要）
- **正答だけ長く・詳しく・具体的にしない**。4つの選択肢の**長さ・粒度・構造をそろえる**。
  - 正答が突出して長いとき：誤答を同じ構造・長さの「もっともらしい誤り」に充実させる、または正答を意味を保って簡潔化する。
  - 順序問題は「同じステップの順序違い」、条件問題は「同じ要素の組合せ違い」、対象が固有名なら**4つとも固有名**にそろえる（例: 火山の例は全部火山名）。
- **誤答は同じ分野内のもっともらしい誤りにする**。別分野の語・絶対にありえない選択肢・露骨に変な選択肢は禁止。
  例（NG→修正済み）: 火山噴出物の誤答に「P波・S波・断層」「示相化石…」/ 震度の誤答に「震度は岩石の色」「マグニチュードはP波の名前」/ 初期微動継続時間の誤答に「必ず0秒になる」「震度と同じになる」。
- **選択肢内のカッコ書きは原則削除**（補足・言い換え）。ただし次は残す：略語の定義（例「GIS（地理情報システム）」）、記号の説明（「◎（二重丸）」）、数式の括弧、意味の特定に必要な値（「桃色（赤色）」「白色の固体（塩化ナトリウムの結晶）」）。
- **「だけ」「のみ」を選択肢に入れない**（例「気体だけ」→不可）。
- **不自然・場違いな選択肢を入れない**（文脈に合わない「真空」等。代わりに自然な誤りを使う）。
- 重複選択肢禁止。`correctIndex` は範囲内（0〜3）。

### B-2. 問題文
- **問題文から正答が容易に予測できる表現を避ける**（問題文に答えの語がそのまま入っている等）。
  例:「鏡筒が上下に動く顕微鏡は？」→鏡筒上下式（語が一致）/「ばねの弾性で力をはかる道具は？」→ばねばかり。語を言い換えるか、用途・性質で選ばせる形にする。
- **手順・操作だけを問う設問は削除対象**になりやすい（「最初に行う操作は？」等。価値が低ければ削除）。
- 図・グラフ・装置に依存する問題は、必要な数値・条件を問題文に明記してテキストだけで解ける形に。
- 計算問題は**検算**してから出す。

### B-3. その他
- 難易度（basic/standard/advanced）は維持。explanation は修正内容と整合させる。
- quiz を削除する場合も **id は振り直さない**。

## 適用後の自己チェック（必須）
- JSON 破損なし、`flashcards`/`quiz.questions` 以外のフィールド不変。
- 4択：選択肢4つ・重複なし・`correctIndex` 範囲内、「だけ/のみ」不使用。
- 長さ突出（正答が誤答平均の約1.7倍以上かつ+8字以上）が解消。
- フラッシュカード：手順・例・あいまい・噛み合わない・一意でない が残っていない。
