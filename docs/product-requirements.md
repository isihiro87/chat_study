# プロダクト要求定義書 (Product Requirements Document)

## プロダクト概要

### 名称
**まるっとスタディ** - 中学生のためのスマホ学習サイト

### プロダクトコンセプト
- **スマホで気軽に**: 通学中やスキマ時間に、スマホでサクッと学習できる
- **シンプルで迷わない**: 情報を詰め込みすぎず、やることが一目でわかる
- **飽きない仕組み**: 動画、フラッシュカード、クイズなど複数の学習方法で楽しく続けられる

### プロダクトビジョン
中学生が「勉強って面倒くさい」と感じる壁を取り除く学習サイトを目指す。
難しそうに見えないシンプルなデザインで、スマホを開いたらすぐに学習を始められる。
動画で理解し、フラッシュカードで覚え、クイズで確認する。この3ステップで効率的に知識を定着させる。
「ちょっとやってみようかな」と思える気軽さを大切にする。

### 目的
- 中学生がスマホで手軽に学習できる環境を提供する
- 難しそう・面倒くさいという学習への心理的ハードルを下げる
- 複数の学習方法（動画・フラッシュカード・クイズ）で効果的な知識定着を支援する
- まずは歴史科目から開始し、段階的に他科目へ展開する

## 成功指標（KPI）

### プライマリーKPI
- 週間アクティブユーザー（WAU）: 50人（公開後1ヶ月）
- 1セッションあたりの平均滞在時間: 5分以上
- クイズ完了率: 70%以上（クイズを開始したユーザーのうち、最後まで回答した割合）

### セカンダリーKPI
- フラッシュカード1セット完了率: 80%以上
- 動画視聴完了率: 60%以上
- リピート率: 40%以上（1週間以内に再訪問するユーザーの割合）

### 測定方法
- Google Analytics等のアクセス解析ツールで計測
- ユーザーの個人情報は収集しない（プライバシー配慮）

## 機能要件

### コア機能（MVP）

#### 1. トップページ（科目選択）

**ユーザーストーリー**:
中学生として、学びたい科目をすぐに見つけるために、シンプルな科目選択画面が欲しい

**受け入れ条件**:
- [ ] 科目が大きなカード形式で表示される（タップしやすいサイズ）
- [ ] 最初は「歴史」のみ表示（今後拡張予定であることを示唆）
- [ ] スマホで見たときに1画面で全体が把握できる
- [ ] 読み込み時間が2秒以内

**優先度**: P0（必須）

#### 2. 単元選択ページ

**ユーザーストーリー**:
中学生として、学びたい単元を簡単に選ぶために、わかりやすい単元一覧が欲しい

**受け入れ条件**:
- [ ] 歴史の単元がリスト形式で表示される
- [ ] 各単元にはアイコンまたはイラストがついている
- [ ] 単元名は短く、難しい言葉を避ける（例：「古墳時代」→「古墳時代 〜大きなお墓を作った時代〜」）
- [ ] スクロールで全単元を確認できる
- [ ] MVP段階では「古代〜平安時代」の範囲を対象とする

**優先度**: P0（必須）

#### 3. 学習モード切り替え（タブUI）

**ユーザーストーリー**:
中学生として、その時の気分や目的に合わせて学習方法を選ぶために、簡単に切り替えられるタブが欲しい

**受け入れ条件**:
- [ ] 画面下部に固定タブバーを配置
- [ ] タブは「まなぶ」「動画」「カード」「クイズ」の4つ
- [ ] 現在選択中のタブが視覚的にわかる
- [ ] タブ切り替え時のページ遷移が0.3秒以内
- [ ] アイコン+テキストで直感的にわかる

**優先度**: P0（必須）

#### 4. 説明資料ページ（まなぶタブ）

**ユーザーストーリー**:
中学生として、その単元の内容を理解するために、見やすい説明資料が欲しい

**受け入れ条件**:
- [ ] 文章は短く、1段落3行以内を目安
- [ ] 重要なキーワードは太字またはハイライト
- [ ] イラストや図を効果的に配置
- [ ] スマホでの読みやすさを考慮したフォントサイズ（16px以上）
- [ ] 文字だけが続かないよう、視覚的な区切りを入れる

**優先度**: P0（必須）

#### 5. 動画ページ（動画タブ）

**ユーザーストーリー**:
中学生として、動画で楽しく学ぶために、見やすい動画プレイヤーが欲しい

**受け入れ条件**:
- [ ] 横型動画：画面幅いっぱいに表示（16:9比率）
- [ ] 縦型動画：スマホ画面に最適化（9:16比率）
- [ ] 縦型動画は上スワイプで次の動画へ（TikTok/リール風UI）
- [ ] 動画の再生/一時停止がワンタップで可能
- [ ] 動画の読み込み中はローディング表示
- [ ] 動画タイトルと簡単な説明文を表示
- [ ] MVP段階ではYouTube埋め込みまたは外部動画サービスを利用

**優先度**: P0（必須）

#### 6. フラッシュカードページ（カードタブ）

**ユーザーストーリー**:
中学生として、重要な用語を覚えるために、めくって確認できるフラッシュカードが欲しい

**受け入れ条件**:
- [ ] カードをタップすると表裏が切り替わる（アニメーション付き）
- [ ] 表面：問題または用語
- [ ] 裏面：答えまたは説明
- [ ] 左右スワイプで次/前のカードへ移動
- [ ] 現在のカード番号と総数を表示（例：3/10）
- [ ] 「覚えた」「もう一度」ボタンで仕分け可能
- [ ] 1セット10枚程度を目安

**優先度**: P0（必須）

#### 7. クイズページ（クイズタブ）

**ユーザーストーリー**:
中学生として、学んだ内容を確認するために、サクッとできるクイズが欲しい

**受け入れ条件**:
- [ ] 4択形式のクイズ
- [ ] 1問1画面で表示（スクロール不要）
- [ ] 選択肢をタップすると即座に正誤判定
- [ ] 正解/不正解がわかりやすいフィードバック（色、アイコン、振動）
- [ ] 不正解時は正解を表示
- [ ] 正誤判定後、タップまたは右スワイプで次の問題へ
- [ ] 全問終了後に結果サマリーを表示（正解数/総問題数＋達成感のあるアニメーション）
- [ ] 1セット5〜10問を目安

**優先度**: P0（必須）

### UIデザイン方針

#### スマホファースト
- モバイルビューポートを基準に設計
- タップ領域は最低44px×44px確保
- 片手操作を想定した配置

#### シンプルさの追求
- 1画面に表示する情報は最小限
- 余白を十分に確保
- 色数は3色以内（メイン、アクセント、背景）
- 装飾は控えめに

#### 親しみやすさ
- 角丸デザインでやわらかい印象
- 明るく優しい配色
- 堅苦しい表現を避ける

#### 直感的な操作性

**ジェスチャー操作**
- 左右スワイプ：次/前のコンテンツへ移動（カード、クイズ）
- 上スワイプ：次の動画へ（縦型動画エリア）
- タップ：カードをめくる、選択肢を選ぶ

**視覚的なガイド**
- 初回のみスワイプを促すアニメーション表示（指が動く図）
- 「次へ」の方向を示す矢印やドットインジケーター
- 操作可能な要素は影やハイライトで強調

**進捗の可視化**
- 画面上部に薄いプログレスバー（今どこまで進んだか）
- ドットナビゲーション（カード：●○○○○）
- 完了時のチェックマークアニメーション

**即時フィードバック**
- タップ時に軽い拡大アニメーション
- 正解時：緑色＋○アイコン＋軽い振動（対応端末）
- 不正解時：赤色＋×アイコン
- カードめくり時：即時切り替え（アニメーションなし）

**迷わない導線**
- 学習の流れを自然に誘導（まなぶ→動画→カード→クイズ）
- 各タブに未完了バッジを表示（やることがわかる）
- 「次はこれ！」のおすすめ表示で次のアクションを提案

### 将来的な機能（Post-MVP）

#### 他科目の追加
数学、英語、理科、国語への展開

**優先度**: P2（できれば）

#### 学習進捗の記録
どの単元を学習したか、クイズの成績などを保存

**優先度**: P1（重要）

#### お気に入り機能
よく見る単元やカードをブックマーク

**優先度**: P2（できれば）

#### ダークモード
目に優しい暗い配色への切り替え

**優先度**: P2（できれば）

## 非機能要件

### パフォーマンス
- 初回ページ読み込み: 3秒以内（3G回線でも5秒以内）
- タブ切り替え: 0.3秒以内
- 画像・動画の遅延読み込み対応

### ユーザビリティ
- 初めて訪れたユーザーが、説明なしで操作方法を理解できる
- 戻るボタンや現在地表示で迷子にならない導線
- 文字サイズは16px以上を基本とする

### アクセシビリティ
- 色のコントラスト比4.5:1以上を確保
- タップ領域44px以上
- 画像には代替テキストを設定

### 信頼性
- 主要ブラウザ（Chrome、Safari、Edge）の最新2バージョンで動作
- オフライン時はエラーメッセージを表示（将来的にはオフライン対応も検討）

### セキュリティ
- HTTPS通信を使用
- ユーザーの個人情報は収集しない（ログイン機能なし）
- 外部埋め込みコンテンツは信頼できるソースのみ使用

### スケーラビリティ
- 科目・単元の追加が容易な構造
- コンテンツ（説明文、クイズ、カード）はデータとして分離

## スコープ外

明示的にスコープ外とする項目:
- ユーザー登録・ログイン機能（MVP段階）
- 学習進捗の保存（MVP段階、ローカルストレージでの簡易保存は検討）
- SNS共有機能
- コメント・掲示板機能
- 課金・有料コンテンツ
- ネイティブアプリ化（Webサイトとして提供）
- 歴史以外の科目（MVP段階）
