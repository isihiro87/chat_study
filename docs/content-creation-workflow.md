# コンテンツ作成・品質改善ワークフロー

## 概要

分析ファイル（`docs/content-analysis-*.md`）をベースに、スキルを使ったコンテンツ作成→フィードバック→分析更新のサイクルを回し、品質を継続的に向上させる。

---

## 全体フロー

```
┌──────────────────────────────────────────────────────────────┐
│                    継続的品質改善サイクル                       │
│                                                               │
│   ① 分析ファイル参照         /prepare-content <教科ID>        │
│       docs/content-analysis/{教科}.md                         │
│       ↓                                                       │
│   ② コンテンツ作成           /improve-from-pdf (Phase 0連携)  │
│                              /generate-content (Step 0連携)   │
│                              /generate-quiz                   │
│       ↓                                                       │
│   ③ レビュー                 /review-content <トピックパス>   │
│       定量・定性チェック + ユーザー確認                        │
│       ↓                                                       │
│   ④ フィードバック反映       /apply-feedback <教科ID> <内容>  │
│       コンテンツ修正 + 分析ファイル更新 + スキル改善           │
│       ↓                                                       │
│   ①に戻る                                                     │
└──────────────────────────────────────────────────────────────┘
```

### 登録済みスキル一覧

| スキル | 種別 | ワークフローの位置 |
|--------|------|-------------------|
| `prepare-content` | skill | ① 作成準備 |
| `improve-from-pdf` | skill | ② 作成（Phase 0で分析ファイル自動参照） |
| `generate-content` | command | ② 作成（Step 0で分析ファイル自動参照） |
| `generate-quiz` | command | ② 作成 |
| `english-content` | skill | ② 作成（英語専用ガイドライン） |
| `review-content` | skill | ③ レビュー |
| `apply-feedback` | skill | ④ フィードバック反映 |

## ステップ詳細

### ① 分析ファイルの参照（作業前）

コンテンツ作成前に、対象教科の分析ファイルを確認する。

| 教科 | 分析ファイル | 教科専用スキル |
|------|-------------|---------------|
| 数学 | `docs/content-analysis/math.md` | — |
| 理科 | `docs/content-analysis/science.md` | — |
| 歴史 | `docs/content-analysis/history.md` | — |
| 地理 | `docs/content-analysis/geography.md` | — |
| 英語 | `docs/content-analysis/english.md` | `english-content` |

**確認すべきポイント:**
- 教材タイプ別の構造・パターン
- ID命名規則
- 改善案（未対応のものがあれば優先的に取り組む）
- 過去のフィードバック履歴（後述の「フィードバックログ」セクション）

### ② コンテンツ作成（スキル活用）

#### パターンA: PDFワークブックからの一括拡充

**最も効率的。PDFがある場合はこれを第一選択肢にする。**

```
/improve-from-pdf <PDFパス> <トピックディレクトリ>
```

- Phase 1〜6の自動フローでクイズ・コンテンツを一括生成
- 対象: quiz/（3ファイル）、chat.ts拡充、flashcards増量、examples追加

#### パターンB: contents.mdからのフル生成

**新規トピック作成時。**

```
/generate-content <トピックフォルダパス>
```

- contents.mdを元にchat.ts + index.tsをフル生成
- contents.mdがなければテンプレート作成→ユーザー記入→再実行

#### パターンC: クイズのみ追加・更新

```
/generate-quiz <contents.mdパス>
```

- 既存コンテンツのクイズ部分のみを更新
- flashcardsは変更しない

#### パターンD: 英語コンテンツ（専用ガイドライン）

英語の場合、上記のいずれかのスキル実行時に自動的に `english-content` スキルのガイドラインが適用される。特に:
- speakable配列の付与
- 例文先行方式
- 禁止用語チェック
- reorder問題の生成

### ③ レビュー

#### 自動レビュー（スキル活用）

```
/review-docs <生成したファイルパス>
```

- doc-reviewerサブエージェントが詳細レビュー
- 完全性・具体性・一貫性・技術的整合性を評価

#### ユーザー確認

生成結果をユーザーに提示し、以下の観点でフィードバックを依頼:
- **内容の正確性**: 教科書・参考書との整合
- **難易度の適切さ**: 対象学年に合っているか
- **コンテンツ量**: 多すぎ/少なすぎないか
- **表現・口調**: 中学生に伝わるか
- **改善すべき点**: 具体的な修正指示

### ④ フィードバック反映

フィードバックは **2箇所** に反映する:

#### A. コンテンツの即時修正

指摘された問題をその場で修正。

#### B. 分析ファイルの更新（重要）

フィードバックが **今後の他トピック作成にも適用すべき汎用的な知見** の場合、対応する分析ファイルを更新する。

**更新箇所:**

1. **「改善案」セクションの更新**
   - 実施済みの改善案に ✅ マークを追加
   - 新たな改善案を追記
   - 効果がなかった改善案に注記

2. **「フィードバックログ」セクションの追加**（各分析ファイルの末尾）

```markdown
## 5. フィードバックログ

### [日付] フィードバック内容の要約
- **対象**: トピック名・ファイルパス
- **指摘**: 具体的な問題点
- **対応**: 修正内容
- **汎用化**: 今後の作成時に適用すべきルール
```

### ⑤ スキル改善（必要に応じて）

フィードバックが **スキル自体の改善** に値する場合:

| 改善内容 | 更新ファイル |
|----------|------------|
| 英語の表現ルール追加 | `.claude/skills/english-content/SKILL.md` |
| PDF処理の手順改善 | `.claude/skills/improve-from-pdf/SKILL.md` |
| コンテンツ生成の品質基準追加 | `.claude/commands/generate-content.md` |
| クイズ生成のルール追加 | `.claude/commands/generate-quiz.md` |

**スキル更新の判断基準:**
- 同じフィードバックが2回以上出た → スキルに反映すべき
- 教科固有のルール → 分析ファイルに記録
- 教科横断のルール → スキルファイルに反映

## 実践例: 典型的な作業セッション

### 例1: 中2理科の新トピック作成

```
1. ユーザー: 「中2理科の天気の章を作って」

2. Claude:
   - docs/content-analysis/science.md を読み、理科の構造を確認
   - PDFがあれば /improve-from-pdf を実行
   - なければ /generate-content を実行
   - 完了後、ユーザーに確認

3. ユーザー: 「スライドのreasonに具体的な数値がもっと欲しい」

4. Claude:
   - コンテンツを修正
   - docs/content-analysis/science.md の改善案に追記:
     「reasonスライドには可能な限り具体的な数値を含める」
   - 今後の理科コンテンツ作成時に自動的に参照される
```

### 例2: 中1英語のフィードバック反映

```
1. ユーザー: 「be動詞のchat.tsで、否定文の例文が少ない」

2. Claude:
   - chat.tsの否定文セクションに例文追加
   - docs/content-analysis/english.md のフィードバックログに記録
   - english-contentスキルに「否定文は最低3例文」ルール追加を検討
```

## ファイル間の関係図

```
docs/content-analysis/{教科}.md     ←── 品質基準・パターン定義
        ↕ 参照・更新
.claude/skills/*/SKILL.md           ←── 生成ルール・手順定義
.claude/commands/*.md               ←── コマンド定義
        ↓ 実行
src/data/subjects/{教科}/...        ←── 実際のコンテンツ
        ↓ レビュー
docs/content-analysis/{教科}.md     ←── フィードバック蓄積
```

## 品質指標の追跡

各分析ファイルに以下の指標を定期的に更新:

| 指標 | 目標値 | 追跡方法 |
|------|--------|----------|
| トピックカバレッジ | 全トピックにchat + index + quiz/ | Glob検索 |
| フラッシュカード数 | 15〜30枚/トピック（difficulty付き） | index.ts確認 |
| クイズ問題数（index.ts） | 15〜25問/トピック（difficulty付き） | index.ts確認 |
| 一問一答数 | 40〜50問/トピック | ichimondittou.md確認 |
| 大問小問数 | 25〜30問/トピック | structured.md確認 |
| 発展問題数 | 15〜20問/トピック | advanced.md確認 |
| 例題数 | 4〜6題/トピック | index.ts確認 |
| スライド数（理科・地理） | 3〜5セット/トピック | index.ts確認 |
| 難易度バランス | basic 40% / standard 40% / advanced 20% | index.ts確認 |

---

## まとめ

1. **分析ファイルは「生きたドキュメント」** — フィードバックのたびに更新
2. **スキルは「自動化された品質基準」** — 繰り返し出るフィードバックはスキルに反映
3. **サイクルを回すほど精度が上がる** — 分析→作成→FB→分析更新→次の作成に反映
