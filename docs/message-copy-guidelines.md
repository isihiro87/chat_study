# 公式LINE メッセージ文言ガイドライン（正本）

公式LINE「チャットでスタディ」のユーザー向け文言（webhook応答・配信push・flex・AIチャット）の統一基準。
2026-07 の全体監査（.steering/20260718-line-message-consistency/）で確立。**新しい文言を書く・変えるときは必ずここに従う。**

## 1. ブランド・呼称

| 項目 | 正 | 禁止・注意 |
|---|---|---|
| サービス名 | チャットでスタディ | 略称・愛称を作らない（「ちゃすた」等はAIの捏造実例あり）。「ワークでスタディ」等の別ブランド風表記も禁止 |
| AIペルソナ | スタ先生（メインチャット・参考書対話・ムビスタで同一人格） | 「AI先生」「先生（AI）」は廃止済み。名乗りは聞かれたときだけ |
| AIの一人称 | わたし | ぼく/俺に揺らさない |
| 二人称 | きみ（無主語も可） | 「あなた」は避ける |
| 運営の言及 | 「運営の夫婦」「夫婦2人で運営」 | 個人名・塾名は出さない。学歴は「東大大学院」まで（「東大卒」表記禁止） |

## 2. 文体

- **基調はタメ口やわらか**：「〜だよ」「〜してね」「〜しよう」。押しつけない・責めない・絵文字は控えめ（1〜2個、機能的に）。
- **敬体を使ってよいのは、つづもん購入・ライセンス・料金の案内のみ**（保護者が読む前提のため）。「〜です🙏」「〜してください」。
- **エラー文言もタメ口で謝る**：「ごめんね、〜できなかったみたい💦 もう一度試してみてね」。「エラーが発生しました。」のような無機質文は禁止。
- 用語は中学生語彙に言い換える（オンボーディング→最初の登録、フラッシュカード→暗記カード）。

## 3. 機能・ボタンの呼称（現行リッチメニュー準拠）

正: **1問解く / 苦手を復習 / じっくり学ぶ / 出題範囲設定 / 成績・記録 / 設定・サポート**

- 文中でメニューを指すときは必ずこの名前。「テスト範囲設定」「範囲を設定する」「追加で解く」等の別名でメニューを指さない。
- 同一メッセージ内のボタン/チップを指すときは「下の『◯◯』ボタン」と付け、メニューと区別する。
- テキストコマンドの案内は「『設定変更』と送ってね」（送る対象はトーク）。

## 4. 事実の言い方（齟齬の再発防止）

| テーマ | 正しい言い方 | 禁止 |
|---|---|---|
| 配信頻度 | 「はじめは毎日1問、そのあとは週3回（月・水・金）」 | 「毎日届く」と言い切らない |
| 配信時刻 | 朝6時 / 朝7時 / 夕方4時 / 夕方6時 / 夜8時 の5択 | |
| 設定変更 | テキスト「設定変更」＝学年・教科・配信時刻の全再設定（1日1回）。学年・教科のみ＝「設定・サポート」内「学年・教科を変更」（1日3回） | 回数・範囲を混同しない |
| 教科 | 登録は1つだけ（変更であって「追加」ではない） | 「教科を追加できる」 |
| 料金 | 公式LINEの機能は全部無料。別売の問題集「つづもん」だけ有料 | 「すべて無料」とつづもん文脈で言わない。¥680/¥980 は休眠中（docs/operations/premium-dormant-inventory.md） |
| 範囲設定の保存 | 単元は「タップした時点で保存」。「これで決定」は締めの操作 | 手入力「これで決定」に完了断言しない |
| 経過日数（Win-back等） | 実送信日と矛盾する具体日数を書かない（「ちょっとひさしぶり」等の曖昧表現か実日数） | 内部タッチポイント名（day3等）の数字を本文に書く |

## 5. AIチャット固有ルール（aiChatPrompt.ts）

- AIは**設定変更・再開・範囲保存を実行できない**。「変更したよ」「再開したよ」と実行済みのように言わせない（「できないこと」節）。
- 配信された問題の答えは教えずヒント（学校の宿題はOK）。
- 仕様を推測ででっち上げない。分からないことは「お問い合わせ」へ。

## 6. 変更時の検証手順

1. 静的テスト: `npx vitest run functions/src/__tests__`（`aiChatPrompt.test.ts` がプロンプトの正本知識を検査）
2. AIプロンプトを変えたら: **`npm run ai-qa`**（静的テスト＋Gemini実応答のライブeval を一括実行・要 GEMINI_API_KEY）
3. 文言grep: 「毎日決まった時間」「テスト範囲設定」（ボタン名として）「AI先生」等の禁止表現が増えていないか
4. 定期的に実会話を確認: `npx tsx scripts/_line-conversation-snapshot.ts <nowMs> [days]`（read規律遵守・結果は log-snapshots へ）

## 7. 週次QAルーチン（eval定常運用・2026-07-19〜）

毎週1回（月曜推奨）、次の3ステップを回す。Claude Code には「週次AI QAをやって」と言えばよい:

1. `npx tsx scripts/_line-conversation-snapshot.ts $(date +%s%3N) 7` → 直近7日の実会話を取得（約150 reads）
2. スナップショットを読み、AI の誤案内・未対応パターンを洗い出す（誤実行確認・仕様でっち上げ・メニュー名違い等）
3. 見つけたパターンを `scripts/eval-ai-chat.ts` の CASES に追加 → `npm run ai-qa` で全ケース pass を確認 → 必要ならプロンプト修正＋lineWebhook デプロイ

evalケースは削らず蓄積する（回帰防止）。結果と修正は log-snapshots＋steering に記録する。
