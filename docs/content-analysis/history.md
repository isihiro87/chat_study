# 歴史（history）教材作成分析と改善案

## 1. ディレクトリ構成

```
src/data/subjects/history/
├── index.ts
└── eras/
    ├── contents.md               # 中3用参考資料（PDF要約）
    ├── cards.md                  # プレースホルダー
    ├── grade1/                   # 5時代・23以上のトピック
    │   ├── 1-ancient-world/
    │   ├── 2-japanese-origins/
    │   ├── 3-ancient-state/
    │   ├── 4-warrior-kamakura/
    │   └── 5-medieval-world/
    ├── grade2/                   # 9時代・45以上のトピック
    │   ├── 1-early-modern/ ... 9-meiji-late/
    └── grade3/                   # 7時代・56以上のトピック
        ├── 1-ww1-japan/ ... 7-modern-world/
```

**トピック構成:**
```
topics/{topic-id}/
├── index.ts      # Topic定義（explanation, flashcards, quiz）
├── chat.ts       # 対話形式の歴史解説（主要コンテンツ）
└── quiz/

    └── advanced.md       # 発展問題
```

**規模:** 21時代・124トピック以上（grade2は9時代・47トピック）

## 2. 教材タイプ別の作り方

### 2.1 チャット（chat.ts）— 歴史の主要コンテンツ

歴史はチャット形式が最も中心的な学習コンテンツ。

**型:** `HistoryChat`（この型名の由来は歴史教科から）

**キャラクター例:**
```typescript
characters: [
  {
    id: 'researcher',
    name: '研究者',
    emoji: '🧐',
    colorFrom: '#7c3aed',    // 紫系
    colorTo: '#a78bfa',
    expressions: { explaining: '🧐', happy: '😊', excited: '🤩', thinking: '🤔' },
  },
  {
    id: 'student',
    name: '生徒',
    emoji: '👦',
    colorFrom: '#059669',    // 緑系
    colorTo: '#34d399',
    expressions: { curious: '🙋‍♂️', surprised: '😲', thinking: '🤔', happy: '😄' },
  },
]
```

**特徴:**
- キャラクターには「先生」「研究者」「歴史の先生」等の一般名を使用する（**歴史上の人物名は禁止**: 孔子先生、卑弥呼先生等はNG）
- `<ruby>`タグでふりがな（歴史用語に必須）
- `<span class="keyword">`で重要語句ハイライト
- `data-tooltip`属性でホバー定義
- `date`タイプで時代区分マーカー

**コンテンツ要素:**
| タイプ | 用途 | 歴史特有の使い方 |
|--------|------|------------------|
| `date` | 時代マーカー | 「約700万年前〜」等の年代 |
| `narrator` | 時代背景の解説 | 社会状況の描写 |
| `message` | キャラクター対話 | 人物・出来事の解説 |
| `summary-point` | 重要ポイント | 年号・人物・出来事の要約 |
| `quiz` | 確認クイズ | 年代・人物名の確認 |
| `image` | 資料画像 | 地図・年表・資料 |
| `end` | まとめ | 時代の重要事項リスト |

### 2.2 解説（explanation）

```typescript
explanation: {
  sections: [
    {
      title: '猿人の出現',
      content: '約700万年前、アフリカで最初の人類である猿人が現れた...',
      keyPoints: ['約700万年前にアフリカで出現', '二足歩行を開始', ...],
    },
  ],
  // ※歴史にはslidesは基本的に使用しない
}
```

### 2.3 フラッシュカード

**構成:** quiz/ichimondittou.mdの全問題をカバーする量（問題数はトピックごとに異なってよい）

```typescript
{
  id: 'fc1',
  front: '猿人',                           // 用語（裏面に表示）
  back: '約700万年前にアフリカで現れた最古の人類は？',  // 問題（表面に表示）
  explanation: '約700万年前にアフリカで出現し、二足歩行を始めた最初の人類。',
}
```

**特徴:**
- 歴史用語・年号・人物名の暗記に最適化
- front/backの表示が直感と逆（UIではback→front順に表示）
- `explanation`: 補足説明（front/backとは別に、背景知識や覚え方のポイントを1文で記載）

### 2.4 クイズ

**index.ts内:** quiz/フォルダ3ファイルから作れるだけの4択問題を網羅的に変換（問題数はトピックごとに異なってよい）

```typescript
{
  id: 'q1',
  question: '約700万年前にアフリカで現れた最古の人類を何というか？',
  options: ['新人', '原人', '猿人', '旧人'],
  correctIndex: 2,
  explanation: '猿人は二足歩行を始めた最初の人類です...',
}
```

**quiz/ディレクトリ（index.tsのクイズ・フラッシュカードの主要ソース）:**

**⚠️ quiz/フォルダ活用ルール:**
- quiz/フォルダの問題は「追加」ではなく、index.tsのクイズ・フラッシュカードの**主要な出題元**
- 各ファイルの問題を漏れなくindex.tsに反映する
- quiz/フォルダに問題があるのにindex.tsに反映されていないのは**不完全な状態**

- `ichimondittou.md`: 一問一答 → **全Q&Aペアをフラッシュカードに変換 + 4択クイズ化**
- `structured.md`: 大問形式 → **各小問を独立した4択クイズに分解・変換**
- `advanced.md`: 発展問題 → **difficulty: 'advanced' タグ付き4択クイズに変換**

### 2.5 例題（examples）

**歴史では基本的に不使用。** 数学・英語のような段階的問題解法は歴史の学習形態に合わないため。

### 2.6 動画（videos）

**一部トピックで使用:** YouTube動画のリンク

## 3. 作成パイプライン

1. **資料確認**: PDFワークブック・contents.mdで時代の内容把握
2. **Era定義**: `grade{X}/{era}/index.ts` にEra作成
3. **チャット作成**: `topics/{topic}/chat.ts` — 歴史では最も重要
4. **Topic定義**: `topics/{topic}/index.ts` に explanation, flashcards(25〜40枚), quiz(25〜40問)
5. **クイズ拡充**: `topics/{topic}/quiz/` にMarkdown問題集
6. **⚠️ quiz/→index.ts反映**: quiz/フォルダの全問題をindex.tsのクイズ・フラッシュカードに変換（最重要ステップ）
7. **集約登録**: units/index.tsに追加

**歴史特有のポイント:**
- チャットが主要学習手段（対話で時代を追体験）
- フラッシュカードは用語暗記のメイン手段
- 年号・人物・出来事の3軸で知識を整理

## 4. 改善案

### 4.1 年表タイムライン機能の追加

**現状:** `date`タイプで年代を表示するが、全体の時系列が見えない。
**提案:** トピック横断の年表コンポーネントを新設。
- 各トピックの主要イベントをタイムライン上にプロット
- Era間のつながりを視覚的に把握可能に
- TopicContentに `timeline: TimelineEvent[]` を追加

### 4.2 人物データベースの構築

**現状:** 人物情報がチャットやフラッシュカードに分散。
**提案:** 共有の人物辞書を作成し、チャット内の人物名にリンク。
- `data-tooltip`は既に使用されているが、統一された人物データが不足
- 同一人物が複数トピックに登場する場合の一貫性確保

### 4.3 スライド（3タップ形式）の導入

**現状:** 歴史ではスライドを使用していない。
**提案:** 因果関係の理解に3タップスライドが有効:
- Question: 「なぜ鎌倉幕府は滅亡したのか？」
- Reason: 「元寇の負担 → 御家人の不満 → 後醍醐天皇の挙兵」
- Conclusion: 「武士の恩賞不足が幕府崩壊の根本原因」

### 4.4 資料読み取り問題の強化

**現状:** advanced.mdに資料解釈問題があるが、画像資料との連携が弱い。
**提案:**
- 歴史資料（史料、地図、統計）のSVG画像を作成
- クイズに画像付き問題を追加（`image`プロパティ活用）
- 高校入試で頻出の資料読み取り力を強化

### 4.5 キャラクターの多様化ガイドライン

**現状:** キャラクターは時代によって変わるが、統一ガイドラインがない。
**提案:** 時代別のキャラクター設定表を作成:
- 古代: 考古学者 + 生徒
- 中世: 歴史研究者 + 生徒
- 近現代: ジャーナリスト + 生徒
- キャラクターの色は教科内で統一ルールを設定

### 4.6 因果関係マッピング

**現状:** トピック間の因果関係が暗黙的。
**提案:** Topicに `causes: string[]`（原因トピック）と `effects: string[]`（結果トピック）を追加。
- 「なぜこうなったのか」を時代を超えて辿れる学習体験
- 歴史的思考力（因果関係の理解）を鍛える

### 4.7 contents.mdの全学年整備

**現状:** contents.mdは中3のみ。中1・中2のまとまった参考資料がない。
**提案:** 全学年分のcontents.mdを整備し、コンテンツ生成の基盤資料として統一。

### 4.8 比較問題の体系化

**現状:** advanced.mdに比較問題があるが、形式が統一されていない。
**提案:** 比較問題の標準テンプレートを作成:
- 時代比較（鎌倉幕府 vs 室町幕府）
- 人物比較（信長 vs 秀吉 vs 家康）
- 制度比較（荘園制 vs 守護地頭制）

## 5. フィードバックログ

### 2026-05-18 全教科共通：FCの「説明文 → 用語」問いの一意性チェック
- **対象**: 全教科全トピック共通（特に歴史FC）
- **指摘**: `data/content/history/08-edo-establishment/edo-culture-learning.json` fc13 で、back「『解体新書』の出版で中心人物として知られる医師は？」に対し正答が「杉田玄白」だったが、**前野良沢も同条件に該当**するため答えが一意に定まらなかった。同様に `edo-politics-reforms.json` fc4「商業を利用して幕府の収入を増やそうとした政治は？」も、享保の改革と田沼政治のどちらでも該当しうる
- **対応**: fc13 → 「『解体新書』の翻訳・出版に関わり、のちに『蘭学事始』を著した医師は？」（杉田玄白固有の著作を追加）、fc4 → 「老中田沼意次が、株仲間の奨励など商業を利用して…」（老中名と固有施策を追加）
- **汎用ルール**: FCで「front=用語、back=説明文の問い」を作る際は、**説明文だけで term が一意に定まるか**を必ず点検する。同時代に類似業績の人物が複数いる場合（杉田玄白 vs 前野良沢、井伊直弼 vs 阿部正弘 など）や、似た性格の施策が複数の改革に存在する場合（倹約令、商業重視、株仲間関連 など）は、**固有の著作・固有の役職・固有の施策・固有の年代**で必ず絞る

### 2026-05-19 全教科共通：誤答に時代外の用語（現代税・現代制度）を混ぜない
- **対象**: 全教科全トピック共通（特に歴史の制度・税・身分に関わるクイズ）
- **指摘**: `social-classes.json` q7「百姓が収穫した米などを領主に納める税は？」で、誤答に**「営業税」（現代税）**が混じっており、江戸時代の税であることを知らなくても**「現代用語ではない＝古い用語の方が正答」**というメタ推理で消去可能だった
- **対応**: 「営業税」→「諸役」（江戸時代の労役負担）に置換し、運上金・冥加金とともに**すべて江戸時代の税・負担用語**で揃えた
- **汎用ルール**: 歴史の用語クイズでは、**誤答も問題の時代に存在した用語**で揃える。現代用語（営業税・所得税・消費税・教育委員会・国会 など）が混じると時代知識だけで消去可能になるため、その時代に実在した同カテゴリの用語（江戸時代なら諸役・運上金・冥加金・助郷など）で誤答を構成する

### 2026-05-19 全教科共通：誤答の質を「並列構造のカテゴリ内」で揃え、消去法を防ぐ
- **対象**: 全教科全トピック共通（特に「○○とはどのような○○？」型の用語定義クイズ）
- **指摘**: `edo-transport-commerce.json` q2「宿場町とはどのような場所？」で、誤答が「年貢米を集める場所／外国船を取りしまる場所／銀貨をつくる場所」と**カテゴリも文脈も無関係**で、宿場町を知らなくても明らかに消去できる状態だった。同様に q3「大坂から江戸へ酒を運んだ廻船は？」も、誤答（北前船／南蛮船／朱印船）が全て**ルート・性格が違いすぎる**ため消去法が容易
- **対応**: q2は**江戸時代の都市タイプ並列**（城下町／港町／宿場町／門前町）に差し替え、q3は**同じ大坂↔江戸ルートの菱垣廻船を誤答に追加**して、「酒の専門」が樽廻船という詳細を学習対象にした
- **汎用ルール**: 用語定義クイズの誤答は、**正答と同じカテゴリ内の隣接概念**（都市タイプなら他の都市タイプ、廻船なら他の廻船、改革なら他の改革、文化人物なら同時代の他の人物）で揃える。荒唐無稽な誤答は「知らなくても解ける」問題になり学習価値が下がる。正答と誤答の差が**カテゴリの違い**ではなく**カテゴリ内の細部の違い**になることをめざす

### 2026-05-19 全教科共通：正答と問題文の同義語echoを避ける
- **対象**: 全教科全トピック共通（特に「○○とはどのような○○？」型の用語定義クイズ）
- **指摘**: `edo-transport-commerce.json` q12「両替商とはどのような商人？」で、正答が「金貨と銀貨の**交換**を行う商人」だったが、**「両替」と「交換」は直接の同義語**であり、用語の意味を知らなくても正答が推測可能だった
- **対応**: 「交換」→「取引」に変更（取引は両替の上位概念だが直接の同義語ではない）
- **汎用ルール**: 漢字の一致（2026-05-19の鰯ルール）だけでなく、**意味の直接同義（両替↔交換、貿易↔交易、寺子屋↔教育、職人↔手工業 など）**もメタ推測の温床になる。正答を上位概念や周辺概念に言い換え、誤答にも同程度の意味的距離を持たせる

### 2026-05-19 全教科共通：問題文の漢字と正答の語の一致を避ける
- **対象**: 全教科全トピック共通
- **指摘**: `edo-industry-society.json` q2「干鰯とはどのようなもの？」で、正答「いわしを加工した肥料」が **問題文の「鰯」と読み（いわし）で一致** しており、用語の意味を知らなくても漢字を分解するだけで正答が推測できた
- **対応**: 正答を「魚を干して加工した肥料」とぼかし、誤答も「海藻を乾燥させて作った肥料／貝殻を砕いて作った肥料／魚を塩漬けにした保存食」と肥料・海産加工品系で統一
- **汎用ルール**: 「○○とは何か」型の用語定義クイズでは、**問題文に含まれる漢字（特に固有名詞）と正答選択肢の語が一致しないか必ず点検**する。一致している場合は、(1) 正答を上位概念（鰯→魚、菜種→植物、藍→染料原料 など）にぼかすか、(2) 誤答にも同じ漢字／語を散らして相対化する。これは長さ・形式・語尾と並ぶ「メタ推理で正答を見抜ける典型例」

### 2026-05-18 全教科共通：選択肢の語尾も揃える
- **対象**: 全教科全トピック共通
- **指摘**: `edo-politics-reforms.json` q12 で、誤答が「武士の集団／寺院の集団／農民の集団」、正答だけ「商人の同業者組合」と語尾が異なり、**「組合」で終わるのが正答**とメタ推理で見抜けてしまう状態だった
- **対応**: 全選択肢を「○○組合」で統一（武士の身分組合／寺院の宗教組合／農民の自治組合／商人の同業者組合）
- **汎用ルール**: 2026-05-15の「形式・粒度・接尾辞を揃える」ルールの具体例として、**語尾（「組合」「集団」「学問」「政治」「文化」「時代」「改革」など）が正答だけ違うパターン**を必ずチェックする。誤答も同じ語尾で終わるよう揃えること。語尾の不一致は長さ偏り以上にメタ推理されやすい

### 2026-05-18 歴史：江戸三大改革の事実関係（享保→田沼→寛政→天保）
- **対象**: 歴史中2江戸時代トピック
- **指摘**: `edo-politics-reforms.json` fc5 front で「文治政治→元禄期→正徳の治→田沼政治→三大改革」と書かれていたが、**三大改革の最初である享保の改革は田沼政治より前**（享保 1716-1745 → 田沼 1772-1786 → 寛政 1787-1793 → 天保 1841-1843）。さらに「三大改革」を一塊にまとめると田沼の前後関係が不明瞭になる
- **対応**: 「文治政治→元禄期→正徳の治→享保の改革→田沼政治→寛政の改革→天保の改革」と展開
- **汎用ルール**: 江戸時代の政治の流れを示す際は、**三大改革を「享保／寛政／天保」と分けて時系列に並べ、田沼政治を享保と寛政の間に置く**こと。「三大改革」と総称でまとめてしまうと田沼との前後関係が誤解されやすい

### 2026-05-15 全教科共通：4択選択肢の形式・粒度を揃える
- **対象**: 全教科全トピック共通
- **指摘**: `data/content/history/07-early-modern/europe-spread.json` の q13 で、正答だけ「アジアや南米」と複数地域を「や」で並列する形だったのに対し、誤答は「北ヨーロッパ各国 / 北アフリカ各国 / オーストラリア大陸」と単一地域＋接尾辞で、**形式の違いから正答を見抜けてしまう**状態だった
- **対応**: 全選択肢を「○○や○○」の2地域並列形式に統一（北ヨーロッパや北米 / アジアや南米 / 北アフリカや西アジア / オーストラリアや南太平洋）
- **汎用ルール**: 4択クイズの選択肢は **長さだけでなく「形式・粒度・接尾辞」も揃える**。具体的には:
  - 接尾辞（「各国」「大陸」「時代」「文化」等）の有無を統一する
  - 並列構造（「○○や○○」「○○・○○」）の有無を統一する。正答だけ複数要素を含む形にしない
  - 地名・人物名・年代等のカテゴリも揃える（正答が地域名なら誤答も地域名）
  - 長さ偏り防止ルール（2026-03-20）と合わせ、**「メタ情報で正答を推測できるか」をクイズ完成後に必ず点検する**
