# コンテンツ品質基準（全教科共通）

フィードバックから抽出した、全教科・全トピックに適用する品質基準。
コンテンツ作成・レビュー時に必ず参照すること。

## 1. チャット（chat.ts）

### 1.1 クイズの出題レベル
- **直前の説明で教えた解法だけで解ける問題を出す**
- 説明していないステップ（例: ルートを外す、因数分解する等）を暗黙に要求しない
- summary-pointの直後には必ずquizを配置する

### 1.2 対話の構成
- セリフが6〜7つ以上連続する場合は、画像・whiteboard・summary-pointを挟む（4つ程度はOK）
- 挟める適切なコンテンツがない場合は無理に入れなくてよい

### 1.3 用語
- 未知の可能性がある用語には必ず説明をつける
- 学習指導要領で一般に教えない専門用語は避ける（例: 「完全平方数」→「整数の2乗」）

## 2. クイズ（index.ts quiz + chat.ts quiz 共通）

### 2.1 解説の書式
- 数式の後に説明文が続く場合は `\n` で改行を入れる
- 良い例: `'$\\sqrt{18} = 3\\sqrt{2}$\\n$9 = 3^2$ を外に出すよ。'`
- 悪い例: `'$\\sqrt{18} = 3\\sqrt{2}$。$9 = 3^2$ を外に出すよ。'`

### 2.2 選択肢
- 選択肢に「〜のまま」等の不自然な日本語表現を使わない
- 正答の位置（A/B/C/D = correctIndex 0/1/2/3）は均等に分散させる（各25%前後）
- 作成後に正答分布を確認し、偏りがあれば選択肢の順序を入れ替えて調整する

### 2.3 出題元との整合性
- quiz/フォルダの問題（ichimondittou.md, structured.md, advanced.md）を主要な出題元とする
- quiz/フォルダに問題があるのにindex.tsに未反映は不完全な状態

## 3. フラッシュカード

### 3.1 回答と解説の分離
- `front`: 回答のみ（簡潔に）
- `back`: 問題文
- `explanation`: 途中式・解説・補足（任意だが数学では必須）
- `hint`: ヒント（任意）

### 3.2 数学固有ルール
- 数学フラッシュカードでは必ず `explanation` フィールドを使う
- front に途中式や解説を混ぜない
- 「工夫して計算すると？」系の問題では、工夫した方法を解説の先頭に記載する

### 3.3 用語
- 学習指導要領で一般に教えない専門用語はフラッシュカードのback（問題文）にも使わない
- 必要な場合は平易な言い換えを使う（例: 「完全平方数」→「整数の2乗の数」）

## 4. 難易度タグ

- 全フラッシュカード・全クイズに `difficulty` を付与する
- バランス目安: basic 40% / standard 40% / advanced 20%
