# Instagram → 公式LINE 誘導設計

中学生向け Instagram アカウント（中1 / 中2）から、公式LINE「チャットでスタディ」へ誘導するための運用設計。

- **対象 IG アカウント**: 中1 向け / 中2 向け（各1アカウントで別運用）
- **誘導先 LINE アカウント**: 既存の公式LINE 1本に集約（学年は LINE 内 onboarding で分岐）
- **主目的**: Instagram で興味喚起 → 公式LINE友だち追加 → 初回設定 → 1問回答完了
- **想定 KPI**: IG リーチ → LINE 友だち追加 CTR 1〜3% / 友だち追加 → 1問回答完了率 50%+

> 注意: Instagram 側の投稿リンク仕様は変わりやすい。通常の投稿・リール本文の外部URLはクリックできない前提で設計し、ストーリーリンクスタンプ / プロフィールリンク / DM を主導線にする。

---

## 1. 前提整理

### Instagram の導線制約

| 投稿形式 | リンク可否 | 主導線 | 役割 |
|---|---:|---|---|
| Story | 可 | リンクスタンプ | 最短でLINE追加へ送る |
| Feed Post（画像/カルーセル） | 原則不可 | プロフィールリンク / コメント自動DM | 保存・教育・信頼形成 |
| Reels | 原則不可 | プロフィールリンク / コメント自動DM | 認知獲得・軽い参加 |
| コメント欄 | URL記載はできるが主導線にしにくい | キーワードコメント → DM | 行動ハードルを下げる |
| プロフィール | 可 | 常設リンク | 全投稿の受け皿 |
| DM | 可 | LINE追加URL / LP URL | コメント反応者への送客 |

### LINE アカウントを 1 本に集約する理由

- onboarding で「学年」を聞いて grade を保存しているため、LINE アカウントを学年別に分ける必要がない
- 毎日1問の配信内容も grade で出し分け済み
- LINE 公式アカウントの運用・メッセージ通数・リッチメニュー管理を 1 本化できる
- Instagram は「学年別の認知獲得」、LINE は「CRM / 学習継続」に役割を分ける
- 流入元の識別は `src` パラメータと user doc / funnel event で実現する

### 訴求の切り分け

Instagram の初回CTAは **「公式LINEで無料の毎日1問を受け取る」** を基本にする。  
いきなり「7日間無料トライアル」を前面に出すと、友だち追加前の心理ハードルが上がるため。

- 初回接点: 公式LINE追加、毎日1問、テスト範囲に合わせた出題
- LINE追加後: 初回設定、1問回答、必要に応じてプレミアム案内
- プレミアム訴求: 「もっと解きたい」「苦手復習を増やしたい」タイミングで出す

## 2. 投稿フォーマット別 CTA 設計

### 2-1. Stories（最優先・週2〜3本）

5フレーム構成を基本にする。Story はリンクスタンプで直接遷移できるため、最もLINE追加に近い導線。

| フレーム | 役割 | 中1 例 | 中2 例 |
|---|---|---|---|
| 1. フック | 自分ごと化 | 「中学初の定期テスト、何から始めればいい？」 | 「中2の中だるみ、ここで内申に出る」 |
| 2. つまずき | 共感 | 「ワークは開いたけど、何を覚えればいいか分からない」 | 「部活で疲れて、テスト前だけ詰め込む」 |
| 3. 解決 | 小さな行動 | 「毎日1問だけLINEに届く」 | 「苦手を1問ずつつぶす」 |
| 4. 信頼 | 具体性 | 「5教科・263トピックから出題」 | 「テスト範囲に合わせて復習」 |
| 5. CTA | 友だち追加 | 「今日の1問をLINEで受け取る」 | 「中2の復習問題をLINEで受け取る」 |

**リンク先**:

```text
https://line.chatstudy.jp/welcome?src=ig-g1-story-YYYYMMDD
https://line.chatstudy.jp/welcome?src=ig-g2-story-YYYYMMDD
```

**ハイライト常設**:

- `無料で試す`
- `使い方`
- `テスト対策`
- `保護者の方へ`

### 2-2. Feed Post（週1〜2本）

Feed は保存される教材型コンテンツに寄せる。キャプション内URLを主導線にせず、コメントキーワードとプロフィールリンクへ流す。

**投稿テーマ例**:

- 中1: 「be動詞で間違える3パターン」「最初の定期テストで落としやすい理科」
- 中2: 「一次関数の式で詰まる原因」「連立方程式の文章題で見るべき言葉」

**キャプション末尾テンプレ**:

```text
続きの問題は公式LINEで無料配信しています。

コメントに「中1」/「中2」と書いてくれた人には、
LINE追加ページをDMで送ります。

プロフィールのリンクからも追加できます。
```

**固定コメント**:

```text
今日の続きは公式LINEで配信中です。
コメントに「中1」/「中2」で案内をDMします。
プロフィールのリンクからも追加できます。
```

### 2-3. Reels（週2〜3本）

Reels は 15〜30 秒で「気づき」と「参加」を作る。完璧な解説より、コメントしたくなる未完了感を残す。

| 秒数 | 内容 |
|---:|---|
| 0-3s | フック字幕。「中1最初のテストで差がつく問題」「中2でここを落とすと中3で困る」 |
| 3-15s | 1問・1テーマだけ見せる |
| 15-25s | 解き方の一部、または間違えやすい選択肢 |
| 25-30s | CTA。「続きはコメントに『中1』/『中2』で送ります」 |

**ピン留めコメント**:

```text
続きの問題と解説は公式LINEで配信中。
「中1」/「中2」とコメントしてくれた人にDMで送ります。
プロフィールのリンクからも追加できます。
```

### 2-4. コメント欄からの誘導

コメント欄は「URLを貼る場所」ではなく「反応を作る場所」として使う。

**推奨キーワード**:

| アカウント | 基本キーワード | 単元別キーワード |
|---|---|---|
| 中1 | `中1` / `解説` / `LINE` | `英語` / `数学` / `理科` / `社会` |
| 中2 | `中2` / `解説` / `LINE` | `一次関数` / `連立` / `電流` / `歴史` |

**手動返信テンプレ**:

```text
コメントありがとう！
続きの問題はプロフィールの公式LINEから受け取れます。
DMでも案内を送ります。
```

**DM自動応答テンプレ**:

```text
コメントありがとうございます。

中学生向けに、毎日1問の復習問題を公式LINEで配信しています。
テスト範囲に合わせた出題設定もできます。

▼公式LINEを追加する
https://line.chatstudy.jp/welcome?src=ig-g1-dm-YYYYMMDD
```

中2アカウントでは `src=ig-g2-dm-YYYYMMDD` にする。

## 3. 学年別の差別化（中1 / 中2）

同じ公式LINEへ送るが、Instagram 上のフックと文脈は分ける。

### 中1 用アカウント

**世界観**: 中学スタートで勉強の型を作る。

| 要素 | 内容 |
|---|---|
| ペルソナの痛み | 入学直後の不安、勉強方法不明、小学校との差、初めての定期テスト |
| 共感ワード | 「最初のテスト怖い」「ワーク終わらない」「何を覚えればいいか分からない」 |
| 約束 | 「毎日1問だけで、勉強を始めるハードルを下げる」 |
| 信頼材料 | 「5教科・263トピック」「テスト範囲に合わせて出題」 |
| 主CTA | 「中1の今日の1問をLINEで受け取る」 |

**プロフィール文案**:

```text
中1の英数理社を毎日1問で復習
最初の定期テストであわてない勉強習慣づくり
無料問題は公式LINEで配信中
↓ 中1の今日の1問はこちら
```

### 中2 用アカウント

**世界観**: 中だるみ脱出、苦手を中3へ持ち越さない。

| 要素 | 内容 |
|---|---|
| ペルソナの痛み | 中だるみ、内申、部活との両立、受験へのぼんやりした不安 |
| 共感ワード | 「やる気出ない」「テスト前だけ詰め込む」「親が塾を考え始めた」 |
| 約束 | 「中2の苦手を1問ずつつぶして、中3前の穴を減らす」 |
| 信頼材料 | 「5教科・263トピック」「苦手復習」「テスト範囲設定」 |
| 主CTA | 「中2の苦手復習をLINEで始める」 |

**プロフィール文案**:

```text
中2の定期テスト対策・苦手復習
英数理社を毎日1問で積み上げ
中3になる前に穴をなくす
↓ 中2の復習問題はこちら
```

## 4. 計測設計（流入元タグ）

LINE 友だち追加URL自体では細かい流入元を保持しづらいため、`WelcomePage` を経由して `src` を保存する。

### 4-1. URL 設計

```text
https://line.chatstudy.jp/welcome?src=ig-{grade}-{format}-{date}
```

| 例 | 用途 |
|---|---|
| `ig-g1-story-20260520` | 中1 ストーリー |
| `ig-g1-post-20260520` | 中1 投稿 |
| `ig-g1-reel-20260520` | 中1 リール |
| `ig-g1-dm-20260520` | 中1 コメント自動DM |
| `ig-g1-bio` | 中1 バイオリンク |
| `ig-g2-story-20260520` | 中2 ストーリー |
| `ig-g2-post-20260520` | 中2 投稿 |
| `ig-g2-reel-20260520` | 中2 リール |
| `ig-g2-dm-20260520` | 中2 コメント自動DM |
| `ig-g2-bio` | 中2 バイオリンク |

### 4-2. 必要な実装（最小）

現状の `WelcomePage` は `?next=` の保存のみで、`?src=` の保存は未実装。

- `WelcomePage` で `?src=` を validation して localStorage に保存
- LINE Login OAuth callback 後、ユーザー作成または初回 user doc 更新時に `users/{uid}.referralSource` として保存
- 追加で `users/{uid}.referralFirstSeenAt` / `referralLastSeenAt` を持つと、初回流入と再訪を分けられる
- `premiumFunnelEvents` に `welcome_view` または既存 eventType 拡張で `source` を含める

実装するまで、Instagram 側の `src` はURL上の設計にとどまり、Firestore での自動集計はできない。

### 4-3. 見たい指標

`src` ごとに以下を見る。

- WelcomePage view 数
- LINE 友だち追加クリック数
- 新規 user doc 作成数
- 初回設定完了数
- 1問回答完了数
- プレミアム案内ページ閲覧数
- trial 開始数
- 7日後の継続・再訪

`src/pages/AdminDashboardPage.tsx` に src ブレイクダウンを追加する想定。

## 7. 次のアクション候補

1. **`src` パラメータ追跡実装**  
   `WelcomePage` → auth callback / user doc → `premiumFunnelEvents` までつなぐ。

2. **ストーリー文案セット作成**
