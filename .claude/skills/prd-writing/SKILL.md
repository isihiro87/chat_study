# PRD作成スキル

このスキルは、高品質なプロダクト要求定義書(PRD)を作成するための詳細ガイドです。

## テンプレートの参照

PRDを作成する際は、以下のテンプレートを使用してください:

```bash
Read('.claude/skills/prd-writing/template.md')
```

## PRDの構成要素

### 1. プロダクトビジョン

**具体的な価値提案を含める**

悪い例:
```
便利なタスク管理ツールを作る
```

良い例:
```
開発者がターミナルから離れずにタスク管理できるCLIツール。

提供する価値:
- コンテキストスイッチの削減(GUI↔ターミナルの切り替えゼロ)
- 作業効率の向上(マウス操作不要で平均30%の時間短縮)
- 自動化との連携(シェルスクリプトに組み込み可能)
```

### 2. ターゲットユーザー(ペルソナ)

**具体的に定義**

必須要素:
- 年齢、職業、経験年数
- 現在の課題
- 期待する解決策
- 技術スタック
- 1日の典型的なワークフロー

### 3. 機能要件

**ユーザーストーリー形式で記述**

フォーマット:
```
### [機能名]

ユーザーストーリー:
[ユーザー]として、[目的]のために、[機能]が欲しい

受け入れ条件:
- [ ] 条件1(測定可能)
- [ ] 条件2(測定可能)

優先度: P0(必須) / P1(重要) / P2(できれば)
```

### 4. 非機能要件

**測定可能な形で記述**

例:
```
- パフォーマンス
  - コマンド実行時間: 100ms以内
  - タスク一覧表示: 1000件まで1秒以内

- ユーザビリティ
  - 新規ユーザーが5分以内に基本操作を習得できる
  - ヘルプコマンドで全機能を確認できる

- 信頼性
  - データ損失ゼロ(自動バックアップ)
  - エラー発生時のロールバック
```
