# PDF分析からのコンテンツ改善スキル

PDFワークブック（市販問題集など）を分析して、既存トピックのコンテンツを大幅に改善するためのワークフローです。

## 使用方法

```
/improve-from-pdf <PDFパス> <トピックディレクトリ>
```

例:
```
/improve-from-pdf src/data/subjects/science/units/grade3/1-chemistry/topics/3年理科ワーク1章.pdf src/data/subjects/science/units/grade3/1-chemistry/topics/
```

## 全体フロー

```
Phase 1: PDF分析・問題作成
    ↓
Phase 2: コンテンツ拡充（チャット・フラッシュカード・クイズ・例題）
    ↓
Phase 3: トピック分割（チャット8分以内）
    ↓
Phase 4: 図の作成（SVG + プロンプト）
    ↓
Phase 5: ビルド検証・レジストリ再生成
    ↓
Phase 6: コミット・プッシュ
```

---

## Phase 1: PDF分析・問題作成

### 1.1 PDF読み込み

- `poppler-utils` がなければインストール（`apt-get install -y poppler-utils`）
- `pdftoppm` でページごとにPNGに変換し、Read toolで画像として読み取る
- スキャンPDFはテキスト抽出できないため、必ず画像として読む

### 1.2 トピック構造の確認

- 対象ディレクトリ内の既存トピック一覧を確認
- 各トピックの `index.ts` と `chat.ts` を読んで現在のコンテンツ量を把握

### 1.3 クイズMDファイルの作成

各トピックに `quiz/` フォルダを作り、以下の3ファイルを作成:

| ファイル | 内容 | 目安問題数 |
|---------|------|----------|
| `ichimondittou.md` | 一問一答（Q&A形式） | 40-50問 |
| `structured.md` | 大問-小問構成（配点つき） | 大問5、小問25-30 |
| `advanced.md` | 発展問題（計算・論述） | 発展5、小問15-20 |

#### 著作権配慮ルール
- PDFの問題文・構成をそのまま使わない
- 同じ知識を問うが、独自の問題文・選択肢・実験設定で作成
- 図や表のレイアウトも変える

#### 問題作成の品質基準
- 中学生が理解できる日本語で書く
- 各問題に明確な答えと解説をつける
- セクション見出しで分野ごとにまとめる
- 既存のクイズ・フラッシュカード・例題と類似の問題がすでにある場合は追加不要

## Phase 2: コンテンツ拡充

### 2.1 ギャップ分析

作成したクイズ（ichimondittou.md, structured.md）の知識を既存コンテンツと比較し、不足を特定する。

チェックポイント:
- [ ] クイズで問われる用語がフラッシュカードに含まれているか
- [ ] クイズの実験・反応がチャット解説で説明されているか
- [ ] クイズの化学式・反応式が例題でステップ解説されているか
- [ ] 4択クイズが主要知識をカバーしているか

### 2.2 chat.ts の拡充

**不足している知識をチャット形式の対話で追加する。**

ルール:
- 既存のキャラクター設定（teacher/student）を踏襲
- 難しい漢字にはrubyアノテーションをつける
- 各セクション末にsummary-pointを配置
- 重要語句は `<span class="keyword">` で囲む
- 1セクション（dateブロック）あたりメッセージ4-8件を目安

### 2.3 index.ts の拡充

| コンテンツ | 目安数 | 基準 |
|-----------|--------|------|
| フラッシュカード | 15-25枚 | クイズで問われる全用語をカバー |
| 4択クイズ | 8-12問 | 一問一答の主要問題パターンをカバー |
| 例題（ステップ解説） | 4-6問 | 典型問題の解き方を段階的に説明 |
| 解説セクション | 4-7 | チャットのdateセクションに対応 |

### 2.4 例題のステップ設計

- ステップ数は3-6（問題の複雑さに応じて柔軟に）
- 各ステップに `title`, `content`, `highlight` をつける
- highlightには「その手順のキーとなる式・用語」を入れる
- 最終ステップで全体の答えを確認

### 2.5 並行実行

トピックが複数ある場合、Agentツールで並行して拡充を実行する。

## Phase 3: トピック分割

### 3.1 チャット時間の計測

各トピックのchat.tsのアイテム数を数える:

```bash
grep -c "type: 'message'\|type: 'date'\|type: 'summary-point'\|type: 'quiz'\|type: 'image'" chat.ts
```

**目安**: 1アイテム ≒ 12秒。35アイテム ≒ 7分、40アイテム ≒ 8分。

### 3.2 分割判定

| アイテム数 | 対応 |
|-----------|------|
| ~40以下 | 分割不要 |
| 41-70 | 2分割 |
| 71以上 | 3分割 |

### 3.3 分割の実行手順

1. **自然な分割点を決定**: chat.tsの `type: 'date'` セクションの区切りで分ける
2. **新ディレクトリを作成**: `[番号]-[英語名]/` の形式
3. **chat.ts を分割**: 前半・後半それぞれにcharactersブロックをコピーし、endブロックを各ファイルに配置
4. **index.ts を分割**: explanation, flashcards, quiz, examples, slidesをトピックの関連性で振り分け
5. **IDのリネーム**: 新しいプレフィックスで全IDをユニークにする（例: `sci3-ee-fc1`, `sci3-ai-fc1`）
6. **ユニットindex更新**: `index.ts` のimportとTopics配列を更新
7. **science-chat/index.ts更新**: chatのimportとchatMapを更新
8. **orderの調整**: 全トピックのorder番号が連番になるよう調整

### 3.4 並行実行

分割が複数トピックにまたがる場合、Agentツールで並行実行する。ただし、ユニットindexファイルの競合に注意し、各エージェントが更新前にファイルを読み直すよう指示する。

## Phase 4: 図の作成

### 4.1 図が必要な問題の特定

クイズの問題を見直し、中学生が図なしでは理解しにくい問題を特定する:
- 実験装置の模式図
- 反応のイオンモデル
- 数値スケール（pHなど）
- 色変化の視覚表現
- 構造の模式図（原子、電池など）

### 4.2 SVGで作成できるもの（シンプルな模式図）

各トピックの `quiz/images/` ディレクトリにSVGファイルを作成する。

SVG作成ルール:
- viewBox指定（400-500px幅）
- font-family: sans-serif
- 背景色: `#FAF9F7`（プロジェクトのウォームホワイト）
- 教育的な色使い（赤系: `#E63946`, 青系: `#1D3557`, 緑系: `#2ECC71` など）
- テキストラベルは日本語で
- 凡例や注釈をつける

### 4.3 画像生成が必要なもの（写実的・複雑なもの）

`image-guide.md` にプロンプト一覧をまとめる:
- 実験装置の写真風イラスト
- マイクロプレート実験の俯瞰図
- 実物に近い器具の描写

プロンプトのフォーマット:
```markdown
### プロンプトN: `ファイル名.png`（トピックN用）
**対応問題**: 大問X（参考画像）

中学理科の〇〇の実験イラスト。白い背景。
[詳細な描写]
シンプルで見やすい教育用イラスト。テキスト不要。
```

### 4.4 問題との対応表

`image-guide.md` に以下のテーブルを含める:
- SVG画像一覧（パス・内容・対応問題）
- 画像生成プロンプト一覧（ファイル名・内容・対応問題）

## Phase 5: ビルド検証・レジストリ再生成

### 5.1 TypeScript型チェック

```bash
npx tsc --noEmit
```

エラーがあれば修正する。

### 5.2 トピックレジストリ再生成

```bash
npx tsx scripts/generate-topic-registry.ts
```

### 5.3 最終確認

- [ ] 全トピックのorder番号が連番
- [ ] 全IDがユニーク（重複なし）
- [ ] science-chat/index.tsに全chatが登録されている
- [ ] ユニットindexに全topicが登録されている
- [ ] 古いディレクトリが削除されている（分割した場合）

---

## Phase 6: コミット・プッシュ

```bash
git add [対象ファイル]
git commit -m "feat(教科): [概要]"
git push
```

コミットメッセージに含める情報:
- 何を分析したか（PDFワークブック）
- 作成した問題数
- コンテンツの拡充内容
- トピック分割の結果（分割した場合）
- SVG画像の作成数

---

## チェックリスト

作業完了時に以下を確認:

- [ ] クイズ3タイプ（一問一答・大問小問・発展）が各トピックに作成されている
- [ ] チャット解説がクイズの知識をカバーしている
- [ ] フラッシュカードがクイズの全用語をカバーしている
- [ ] 例題が典型問題パターンをステップ解説している
- [ ] 各チャットが8分以内（40アイテム以下）
- [ ] SVG画像が必要な問題に作成されている
- [ ] 画像生成プロンプトがimage-guide.mdにまとめられている
- [ ] TypeScriptビルドが通る
- [ ] トピックレジストリが再生成されている
