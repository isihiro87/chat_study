# コンテンツレビュースキル

生成したコンテンツが教科別の品質基準を満たしているかを検証し、具体的な改善点を報告する。

## 使用タイミング

- `/generate-content` や `/improve-from-pdf` の **後** に実行
- ユーザーが生成結果の品質を確認したいとき

## 使用方法

```
/review-content <トピックディレクトリパス>
```

例:
```
/review-content src/data/subjects/science/units/grade1/1-biology/topics/1-observation
```

---

## ステップ1: 教科の特定と分析ファイル読み込み

トピックパスから教科を判定し、対応する分析ファイルを読み込む:

```
パスに含まれる教科名:
- /subjects/math/     → docs/content-analysis/math.md
- /subjects/science/  → docs/content-analysis/science.md
- /subjects/history/  → docs/content-analysis/history.md
- /subjects/geography/→ docs/content-analysis/geography.md
- /subjects/english/  → docs/content-analysis/english.md
```

英語の場合、追加で `.claude/skills/english-content/SKILL.md` も読み込む。

---

## ステップ2: 対象ファイルの読み込み

トピックディレクトリ内の以下を読み込む:

- `index.ts` （Topic定義）
- `chat.ts` （チャットコンテンツ、存在する場合）
- `quiz/structured.md` （存在する場合）
- `quiz/ichimondittou.md` （存在する場合）
- `quiz/advanced.md` （存在する場合）

---

## ステップ3: 定量チェック

以下の数値を計測し、目標値と比較する:

| 項目 | 計測方法 | 目標 | 判定 |
|------|----------|------|------|
| フラッシュカード数 | flashcards配列の長さ | 15〜30 | ✅/⚠️/❌ |
| クイズ問題数（index.ts） | quiz.questions配列の長さ | 15〜25 | ✅/⚠️/❌ |
| 難易度タグ付与率 | difficulty付きの問題/全問題 | 100% | ✅/⚠️/❌ |
| 例題数 | examples.examples配列の長さ | 4〜6 | ✅/⚠️/❌ |
| 解説セクション数 | explanation.sections配列の長さ | 4〜7 | ✅/⚠️/❌ |
| スライド数（理科・地理） | explanation.slides配列の長さ | 3〜5 | ✅/⚠️/❌ |
| チャットアイテム数 | content配列の長さ | 25〜40 | ✅/⚠️/❌ |
| 一問一答数 | ichimondittou.mdのQ:行数 | 40〜50 | ✅/⚠️/❌ |
| 大問小問数 | structured.mdの(N)行数 | 25〜30 | ✅/⚠️/❌ |
| 発展問題数 | advanced.mdの(N)行数 | 15〜20 | ✅/⚠️/❌ |

判定基準:
- ✅ 目標範囲内
- ⚠️ 目標の70%以上（許容範囲）
- ❌ 目標の70%未満 or ファイル未作成

---

## ステップ4: 定性チェック

### A. 構造的整合性
- [ ] IDが教科の命名規則に従っているか
- [ ] chatIdがchat.tsのidと一致しているか
- [ ] eraIdが正しい親Eraを指しているか
- [ ] orderが適切か

### B. コンテンツ品質
- [ ] フラッシュカードがクイズの全用語をカバーしているか
- [ ] chat.ts内のクイズとindex.tsのクイズが連動しているか
- [ ] 例題のステップ数が適切か（3〜6ステップ）
- [ ] 解説のkeyPointsが各セクションにあるか
- [ ] **summary-pointの後に必ずquizがあるか（chat.ts）**
- [ ] **セリフが6〜7つ以上連続していないか（画像挿入の検討。4つ程度はOK）**
- [ ] **未知の可能性がある用語に説明があるか**
- [ ] **全フラッシュカード・クイズにdifficulty（basic/standard/advanced）が付与されているか**
- [ ] **quiz/フォルダの問題がフラッシュカード・クイズに十分反映されているか**

### C. 教科固有チェック

**数学:**
- [ ] LaTeX記法（`$...$`）が正しく使われているか
- [ ] whiteboard要素がchat.tsに含まれているか

**理科:**
- [ ] 3タップスライドが含まれているか
- [ ] スライドのvisualプロパティが活用されているか

**歴史:**
- [ ] rubyタグで難読漢字にふりがながあるか
- [ ] date要素で時代区分が明確か

**地理:**
- [ ] 3タップスライドが含まれているか
- [ ] 地理データ（面積・人口等）が正確か

**英語:**
- [ ] speakable配列がchat.tsのメッセージにあるか
- [ ] reorder問題がクイズに含まれているか
- [ ] 禁止用語が使われていないか
- [ ] 例文に和訳がついているか

---

## ステップ5: 用語カバレッジ分析

フラッシュカード・クイズ・チャットの3つのコンテンツで、重要用語がどの程度カバーされているかを分析:

1. フラッシュカードのfront/back から用語リストを抽出
2. クイズ問題の選択肢・解説から用語リストを抽出
3. チャットのkeywordスパンから用語リストを抽出
4. 3つのリストを突合し、カバレッジを報告
