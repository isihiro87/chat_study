# フィードバック反映スキル

ユーザーのフィードバックを3つのレイヤーに反映する:
1. **コンテンツ修正** — 指摘された問題の即時修正
2. **分析ファイル更新** — 汎用的な知見の蓄積
3. **スキル改善** — 繰り返し出るフィードバックの自動化

## 使用タイミング

- ユーザーがコンテンツに対してフィードバックを送ったとき
- `/review-content` 後にユーザーが追加指摘をしたとき

## 使用方法

```
/apply-feedback <教科ID> <フィードバック内容>
```

例:
```
/apply-feedback science 理科のスライドのreasonにもっと具体的な数値を入れてほしい
/apply-feedback english be動詞のchat.tsで否定文の例文が少なすぎる
```

---

## ステップ1: フィードバックの分類

フィードバックを以下の3カテゴリに分類する:

### A. 個別修正（このコンテンツだけの問題）
- 事実誤認、タイポ、ID重複など
- → コンテンツを直接修正して完了

### B. 教科パターン（この教科全体に適用すべき知見）
- 「理科のスライドには数値を入れる」「英語の否定文は3例文以上」など
- → コンテンツ修正 + 分析ファイル更新

### C. 全教科共通（教科横断で適用すべきルール）
- 「フラッシュカードのhintを全教科で追加」「チャットの長さを短くする」など
- → コンテンツ修正 + 該当するスキルファイル（SKILL.md/コマンド.md）を更新

---

## ステップ2: コンテンツの即時修正

フィードバック対象のファイルを特定し、修正を実行する。

### 対象ファイルの特定方法

1. フィードバックに具体的なファイルパスが含まれていれば、そのファイルを修正
2. 教科IDとトピック名から推定:
   ```
   src/data/subjects/<教科ID>/.../<トピック>/index.ts
   src/data/subjects/<教科ID>/.../<トピック>/chat.ts
   ```
3. 不明な場合はユーザーに確認

### 修正後の確認

```bash
npx tsc --noEmit
```

---

## ステップ3: 分析ファイルの更新（カテゴリB・Cの場合）

### 3.1 フィードバックログの追記

対象教科の `docs/content-analysis-<教科ID>.md` の末尾に「フィードバックログ」セクションを追記（初回は新規作成）:

```markdown
## 5. フィードバックログ

### [YYYY-MM-DD] [フィードバック要約（1行）]
- **対象**: [トピック名・ファイルパス]
- **指摘**: [具体的な問題点]
- **対応**: [修正内容]
- **汎用ルール**: [今後の作成時に適用すべきルール]
```

### 3.2 改善案セクションの更新

フィードバックが既存の改善案に関連する場合:
- 実施済み → ✅ マーク追加
- 新規知見 → 新しい改善案として追加
- 既存改善案の修正 → 内容を更新

### 3.3 教材タイプ別セクションの更新

フィードバックが作り方のパターン自体に影響する場合:
- 構成・目安数値の修正
- 新しいルール・制約の追加

---

## ステップ4: スキル改善の判定（カテゴリCの場合）

### 4.1 改善が必要なスキルの判定

| フィードバック内容 | 更新するスキル/コマンド |
|-------------------|----------------------|
| コンテンツ生成の品質基準 | `.claude/commands/generate-content.md` |
| クイズの品質・形式 | `.claude/commands/generate-quiz.md` |
| PDF処理のフロー | `.claude/skills/improve-from-pdf/SKILL.md` |
| 英語固有のルール | `.claude/skills/english-content/SKILL.md` |
| レビュー基準の追加 | `.claude/skills/review-content/SKILL.md` |

### 4.2 スキル更新の実行

該当するスキルファイルを読み込み、適切な箇所にルールを追記する。

**更新ルール:**
- 既存の構造を壊さない（セクションの追加 or 箇条書きの追加）
- 追加したルールに `[YYYY-MM-DD追加]` のタグをつける
- ユーザーにスキル更新内容を報告し、承認を得る

### 4.3 同一フィードバック2回ルール

分析ファイルのフィードバックログを確認し、同種のフィードバックが **2回以上** 出ている場合:
- → スキルファイルへの反映を **強く推奨** する
- ユーザーに「このフィードバックは2回目です。スキルに反映しますか？」と提案

## ステップ5: 結果報告

```markdown
## フィードバック反映結果

### 1. コンテンツ修正
- [修正したファイルと内容]

### 2. 分析ファイル更新
- [更新した分析ファイルと追記内容]
- [更新なしの場合: 「個別修正のため、分析ファイルの更新は不要」]

### 3. スキル改善
- [更新したスキルファイルと追記内容]
- [更新なしの場合: 「教科固有の知見のため、スキル更新は不要」]

### 汎用ルール（今後の全コンテンツに適用）
- [抽出された汎用ルール。なければ省略]
```

---

## 注意事項

- フィードバックの分類に迷った場合はカテゴリBとして扱う（分析ファイルに記録して損はない）
- スキル更新はユーザーの承認を得てから実行する
- 分析ファイルのフィードバックログは時系列で追記（古いものは削除しない）
- 分析ファイルが200行を超えそうな場合、古いフィードバックログを要約して圧縮
