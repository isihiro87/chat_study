# English Content スキル

英語科目のトピックデータ（chat.ts, index.ts）を作成・編集する際に準拠するガイドラインです。

## 使用タイミング

- 英語トピックの新規作成時
- 既存英語コンテンツの修正・改善時
- 英語科目のコンテンツレビュー時

---

## 1. 対象ユーザーの前提

**ターゲット**: 中学1〜3年生（特に英語が苦手〜普通の子）

- 小学校で英語に触れているが、文法用語はほぼ知らない状態でスタート
- 「主語」「動詞」のレベルから丁寧に導入が必要
- 文法用語を使う場合は、必ずその場でやさしく言い換える

---

## 2. 言葉づかいのルール

### 2.1 絶対に使わない用語（全学年共通）

以下の文法用語はそのまま使わない。必要な場合は右側の言い換えを使う:

| NG用語 | 言い換え |
|--------|----------|
| 補語 | 「〜です」の後ろのことば |
| 文頭 | 文のいちばん最初 |
| 述語 | 「どうする」「どんなだ」の部分 |
| 目的語 | 「〜を」「〜に」にあたることば |
| 原形 | もとの形（何もつけない形） |
| 語順 | ことばの並べ方 |
| 人称 | （使わず具体例で示す） |
| 冠詞 | a や the（そのまま示す） |
| 可算名詞 / 不可算名詞 | 「数えられるもの」「数えられないもの」 |

### 2.2 学年別で解禁する用語

**中1（grade: 1）** — 初出時は必ずカッコ書きで補足:
- 主語 →「主語（「だれが」「何が」にあたることば）」
- 動詞 →「動詞（「〜する」「〜です」のことば）」
- 否定文 →「否定文（「〜ではない」の文）」
- 疑問文 →「疑問文（「〜ですか？」のように聞く文）」
- 単数・複数 →「1つのとき」「2つ以上のとき」

**中2（grade: 2）** — カッコ補足なしでOK:
- 主語、動詞、否定文、疑問文
- 時制（「時間を表すこと」と初出で補足）
- 受動態 → 初出は「受け身の文」と補足
- 不定詞 → 初出は「to + 動詞のもとの形」と補足

**中3（grade: 3）** — 基本的な文法用語はそのまま使用OK:
- 関係代名詞、分詞、仮定法 なども初出で1回だけ補足
- 自然な範囲で表現の難易度を上げてよい

### 2.3 「主語の見つけ方」の導入パターン

中1の最初のトピックでは、主語を特定するプロセスを明示する:

```
日本語：「私は 生徒です。」
         ↑ 「だれが？」→ 「私」が主語！

英語では主語をいちばん最初に置くよ：
I am a student.
↑ 主語（私）
```

例題のStep解説でも「まず日本語の『だれが？』を探そう」のように案内する。

## 3. コンテンツ構造

### 3.1 ファイル構成（1トピックあたり）

```
topics/N-topic-name/
├── index.ts    # Topic定義（explanation, flashcards, quiz, examples）
└── chat.ts     # チャットコンテンツ
```

### 3.2 ID命名規則

- トピックID: `eng-` + kebab-case（例: `eng-be-verbs`, `eng-general-verbs`）
- Era ID: `english-grade` + 学年数（例: `english-grade1`）
- フラッシュカードID: `eng-{topic短縮}-fc{番号}`
- クイズID: `eng-{topic短縮}-q{番号}`
- 例題ID: `eng-{topic短縮}-ex{番号}`
- ChatID: `eng-` + kebab-case（トピックIDと同じ）

### 3.3 コンテンツ量の目安

| 要素 | 数量 |
|------|------|
| explanation.sections | 2〜3 |
| flashcards | 8〜12枚 |
| quiz.questions | 5問 |
| examples | 3問 |
| chat scenes (date〜quiz) | 2〜3シーン |

## 4. チャット（chat.ts）ガイドライン

### 4.1 キャラクター設定

必ず2名構成:

**先生（side: 'left'）**
- emoji: `👩‍🏫` / expressions: explaining, happy, encouraging, thinking
- 口調:「〜だよ」「〜なんだ」（親しみやすい）
- 英語の例文は必ず **日本語の意味** をセットで示す

**生徒（side: 'right'）**
- emoji: `👦` / expressions: curious, surprised, thinking, happy, confused
- 口調:「〜ですか？」「〜なんですね！」
- 各シーンで最低1回は質問する
- 素朴な疑問を投げかける:
  - 「え、amとisって何が違うんですか？」
  - 「どうやって使い分けるんですか？」
  - 「覚え方ってありますか？」

### 4.2 英文の扱い

- 英文を出すときは必ず日本語訳をつける
- 例: `I am a student.（私は生徒です。）`
- メッセージ内で英文が複数ある場合はリスト形式にして見やすく
- HTMLタグ:
  - 重要英単語: `<strong>am</strong>`
  - 重要パターン: `<strong>I am 〜.</strong>（私は〜です。）`

### 4.3 content配列の構成

```
date（テーマ）
  → narrator（導入）
  → message × 2-4（先生が説明、生徒が質問、先生が答える）
  → summary-point（要点1行）
  → quiz（理解度チェック1問）

date（次のテーマ）
  → narrator
  → message × 2-4
  → summary-point
  → quiz

end（まとめ4-5点）
```

## 5. 例題（examples）ガイドライン

### 5.1 ステップ設計の原則

**「何を考えればいいか分からない子」でも手順通りに進められる設計**

各ステップで明確な1アクション:
1. 日本語から「だれが？」を探す → 主語を決める
2. 主語に合う動詞の形を選ぶ
3. 残りの部分を英語にする

### 5.2 ステップの書き方

```typescript
{
  title: 'Step 1: 「だれが？」をさがそう',       // 短く、動詞で始める
  content: '「私は 生徒です」→「だれが？」→「私」だね。英語で I だよ。',  // やさしい語りかけ
  highlight: 'I',                                  // このステップの答え
}
```

- title: 「Step N: 〜しよう」の形
- content: 語りかけ口調で、1〜2文
- highlight: そのステップで得られるキーワード or 英語フレーズ

### 5.3 例題の構成パターン（中1）

1. **肯定文をつくる** — 日本語 → 英語の基本変換
2. **否定文をつくる** — 既存の英文を変換
3. **疑問文をつくる＋答え方** — 変換 + 応答

### 5.4 学年別の難易度調整

**中1**: 3ステップ。各ステップは1操作のみ。「だれが？」「どうする？」の日本語からのアプローチ。
**中2**: 3〜4ステップ。文の構造分析を少し含む。
**中3**: 3〜5ステップ。複数の文法事項を組み合わせる問題も可。

## 6. フラッシュカード（flashcards）ガイドライン

### 6.1 表示の仕組み（重要）

FlashcardDeck コンポーネントは **back を表面（最初に見える面）**、**front をフリップ後の裏面** として表示する。

したがって:
- **back**: 問題文（穴埋め形式の文。最初に表示される）
- **front**: 答え（1語〜短いフレーズ。フリップで表示される）
- hint: 考え方のヒント（中1は特に丁寧に）
- explanation: 正解の理由（1文）

### 6.2 データの書き方

```typescript
// FlashcardDeck では back→表面（先に見える）、front→裏面（フリップで見える）
{
  id: 'eng-be-fc1',
  front: 'am',                                    // ← 答え（フリップで見える）
  back: 'I ___ a student.（私は生徒です。）',      // ← 問題（最初に見える）
  hint: '「私 = I」のときのbe動詞は…？',
  explanation: 'I のときは、いつも am だよ。',
}
```

**注意**: front/back の名前と実際の表示が逆なので、必ずコメントで明示すること。歴史科目も同じ規則。

## 7. クイズ（quiz）ガイドライン

- 4択。correctIndexの位置はバラけさせる
- explanation は文法用語を避け、理由をやさしく説明
- 中1: 基本的な形の選択問題が中心
- 中2-3: 文脈を含む問題、誤文訂正なども可

## 8. 品質チェックリスト

コンテンツ作成後、ファイルに書き込む前に確認:

### A. 言葉づかい
- [ ] 「補語」「文頭」「述語」「目的語」など禁止用語を使っていないか
- [ ] 学年に応じた文法用語のみ使用しているか
- [ ] 初出の文法用語にカッコ書き補足があるか（中1）
- [ ] 英文に日本語訳がついているか

### B. 例題の質
- [ ] 各ステップが明確な1アクションか
- [ ] 「だれが？」「どうする？」など日本語からのアプローチがあるか（中1）
- [ ] highlight が各ステップに設定されているか
- [ ] 3パターン（肯定・否定・疑問）が含まれているか

### C. チャットの質
- [ ] 生徒が各シーンで質問しているか
- [ ] 先生の説明が1メッセージ2〜3文以内か
- [ ] 英文に日本語訳がセットになっているか
- [ ] summary-pointが各シーンにあるか

### D. 技術的整合性
- [ ] IDが `eng-` プレフィックスで始まっているか
- [ ] chatId が chat.ts の id と一致しているか
- [ ] english-chat/index.ts に登録されているか
- [ ] TypeScriptコンパイルが通るか
