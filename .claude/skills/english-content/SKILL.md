# English Content スキル

英語科目のトピックデータ（chat.ts, index.ts）を作成・編集する際に準拠するガイドラインです。

## 使用タイミング

- 英語トピックの新規作成時
- 既存英語コンテンツの修正・改善時
- 英語科目のコンテンツレビュー時

---

## 1. 対象ユーザーの前提

**ターゲット**: 中学1〜3年生（特に英語が苦手〜普通の子）

- 小学校で英語に触れているが、文法用語はほぼ知らない状態でスタート
- 「主語」「動詞」のレベルから丁寧に導入が必要
- 文法用語を使う場合は、必ずその場でやさしく言い換える

---

## 2. 言葉づかいのルール

### 2.1 絶対に使わない用語（全学年共通）

以下の文法用語はそのまま使わない。必要な場合は右側の言い換えを使う:

| NG用語 | 言い換え |
|--------|----------|
| 補語 | 「〜です」の後ろのことば |
| 文頭 | 文のいちばん最初 |
| 述語 | 「どうする」「どんなだ」の部分 |
| 目的語 | 「〜を」「〜に」にあたることば |
| 原形 | もとの形（何もつけない形） |
| 語順 | ことばの並べ方 |
| 人称 | （使わず具体例で示す） |
| 冠詞 | a や the（そのまま示す） |
| 可算名詞 / 不可算名詞 | 「数えられるもの」「数えられないもの」 |

### 2.2 学年別で解禁する用語

**中1（grade: 1）** — 初出時は必ずカッコ書きで補足:
- 主語 →「主語（「だれが」「何が」にあたることば）」
- 動詞 →「動詞（「〜する」「〜です」のことば）」
- 否定文 →「否定文（「〜ではない」の文）」
- 疑問文 →「疑問文（「〜ですか？」のように聞く文）」
- 単数・複数 →「1つのとき」「2つ以上のとき」

**中2（grade: 2）** — カッコ補足なしでOK:
- 主語、動詞、否定文、疑問文
- 時制（「時間を表すこと」と初出で補足）
- 受動態 → 初出は「受け身の文」と補足
- 不定詞 → 初出は「to + 動詞のもとの形」と補足

**中3（grade: 3）** — 基本的な文法用語はそのまま使用OK:
- 関係代名詞、分詞、仮定法 なども初出で1回だけ補足
- 自然な範囲で表現の難易度を上げてよい

### 2.3 漢字とルビ（ふりがな）のルール

**漢字を使って、難しい漢字にはルビを振る。**

ひらがなで書くと、読める生徒にとってかえって読みづらくなる。漢字で書き、必要に応じてHTMLの `<ruby>` タグでルビを振ること。

#### 対象

- 小学校高学年〜中学で習う漢字（以上、以外、背、忙しい、短い、縮む、練習、注意、魚、基本、一番 など）
- 文法用語（動詞、否定、短縮形 など）

#### 書き方

```html
<ruby>以上<rt>いじょう</rt></ruby>
<ruby>短<rt>みじか</rt></ruby>い
<ruby>忙<rt>いそが</rt></ruby>しい
<ruby>動詞<rt>どうし</rt></ruby>
<ruby>一般動詞<rt>いっぱんどうし</rt></ruby>
```

#### ルビ不要な漢字

小学校低学年で習う基本的な漢字はルビ不要:
- 人、文、日、月、私、先、生、見、聞、言、前、後、上、下、大、小、中、高、好、食 など

#### NG: ひらがなのまま（やらないこと）

```
❌ 2人いじょう
❌ いちばん最初
❌ せが高い
❌ いそがしい
❌ ちゅうい
```

#### OK: 漢字+ルビ（こうする）

```
✅ 2人<ruby>以上<rt>いじょう</rt></ruby>
✅ <ruby>一番<rt>いちばん</rt></ruby><ruby>最初<rt>さいしょ</rt></ruby>
✅ <ruby>背<rt>せ</rt></ruby>が高い
✅ <ruby>忙<rt>いそが</rt></ruby>しい
✅ <ruby>注意<rt>ちゅうい</rt></ruby>
```

### 2.4 「主語の見つけ方」の導入パターン

中1の最初のトピックでは、主語を特定するプロセスを明示する:

```
日本語：「私は 生徒です。」
         ↑ 「だれが？」→ 「私」が主語！

英語では主語をいちばん最初に置くよ：
I am a student.
↑ 主語（私）
```

例題のStep解説でも「まず日本語の『だれが？』を探そう」のように案内する。

## 3. コンテンツ構造

### 3.1 ファイル構成（1トピックあたり）

```
topics/N-topic-name/
├── index.ts    # Topic定義（explanation, flashcards, quiz, examples）
└── chat.ts     # チャットコンテンツ
```

### 3.2 ID命名規則

- トピックID: `eng-` + kebab-case（例: `eng-be-verbs`, `eng-general-verbs`）
- Era ID: `english-grade` + 学年数（例: `english-grade1`）
- フラッシュカードID: `eng-{topic短縮}-fc{番号}`
- クイズID: `eng-{topic短縮}-q{番号}`
- 例題ID: `eng-{topic短縮}-ex{番号}`
- ChatID: `eng-` + kebab-case（トピックIDと同じ）

### 3.3 コンテンツ量の目安

| 要素 | 数量 |
|------|------|
| explanation.sections | 2〜3 |
| flashcards | 8〜12枚 |
| quiz.questions | 5問 |
| examples | 3問 |
| chat scenes (date〜quiz) | 2〜3シーン |

## 4. チャット（chat.ts）ガイドライン

### 4.1 説明スタイル（最重要）

**「例文先行→かみ砕き説明」スタイルで統一する。**

ルールや文法用語から入るのではなく、まず学年に応じた簡単な例文を提示し、その文をもとに英語が苦手な子にもわかるようにかみ砕いて説明していく。

#### 基本の流れ

1. **先生が例文を提示する** — シンプルな英文＋日本語訳を見せる
2. **生徒が例文の中の気づき・疑問を言う** — 「am ってなんですか？」「s がついてる！」など
3. **先生がその気づきをもとにかみ砕いて説明する** — 例文を分解しながら、なぜそうなるかを解説
4. **追加の例文で比較・強化する** — 2〜3の例文を並べて違いやパターンを見せる
5. **summary-pointで要点をまとめる**

#### NG: ルール先行パターン（やらないこと）

```
❌ 先生:「be動詞には am, is, are の3つがあるよ。I→am、he/she/it→is...」
❌ 先生:「s/es のつけ方のルールはこうだよ。ふつうは s、ch で終わるときは es...」
```

#### OK: 例文先行パターン（こうする）

```
✅ 先生:「まずこの文を見てみよう！ I am a student.（私は生徒です。）」
   生徒:「am ってなんですか？」
   先生:「いい質問！この am が今日覚える be動詞だよ。日本語の『〜です』にあたることばなんだ。」

✅ 先生:「この2つの文をくらべてみよう！ I play tennis. / He plays tennis.」
   生徒:「あっ、下の文は plays になってる！s がついてますね！」
   先生:「よく見つけたね！he/she/it のとき、動詞のおしりに s をつけるんだよ。」
```

#### 例文をくらべさせるテクニック

新しい文法事項を導入するとき、生徒に違いを発見させる:

- **肯定→否定**: `I am happy.` と `I am not hungry.` をくらべる → 「not がついてる！」
- **平叙→疑問**: `You are a student.` と `Are you a student?` をくらべる → 「are が最初にきてる！」
- **主語の違い**: `I play tennis.` と `He plays tennis.` をくらべる → 「s がついてる！」
- **be動詞 vs 一般動詞**: `I am happy.` と `I play tennis.` をくらべる → 「am と play、違う種類だ！」

### 4.2 キャラクター設定

必ず2名構成:

**先生（side: 'left'）**
- emoji: `👩‍🏫` / expressions: explaining, happy, encouraging, thinking
- 口調:「〜だよ」「〜なんだ」（親しみやすい）
- 英語の例文は必ず **日本語の意味** をセットで示す
- 例文を提示するときの語りかけ:
  - 「まずこの英文を見てみよう！」
  - 「じゃあ次の例文も見てみよう。」
  - 「この2つの文をくらべてみよう！」

**生徒（side: 'right'）**
- emoji: `👦` / expressions: curious, surprised, thinking, happy, confused
- 口調:「〜ですか？」「〜なんですね！」
- 各シーンで最低1回は質問する
- **例文の中で変化や違いに気づくリアクション** をする:
  - 「am ってなんですか？」（未知の単語への疑問）
  - 「あっ、s がついてる！」（例文の違いへの気づき）
  - 「not がついてますね！」（変化に気づく）
  - 「are が文のいちばん最初にきてる！」（語順の変化に気づく）
  - 「どうやって使い分けるんですか？」（ルールへの疑問）
  - 「覚え方ってありますか？」（学習法への質問）

### 4.3 英文の扱い

- 英文を出すときは必ず日本語訳をつける
- 例: `I am a student.（私は生徒です。）`
- メッセージ内で英文が複数ある場合はリスト形式にして見やすく
- HTMLタグ:
  - 重要英単語: `<strong>am</strong>`
  - 重要パターン: `<strong>I am 〜.</strong>（私は〜です。）`

### 4.4 content配列の構成

```
date（テーマ）
  → narrator（導入：例文を見ていこう、という誘導）
  → message: 先生が例文を提示
  → message: 生徒が例文の中で気づき・疑問を言う
  → message: 先生がかみ砕いて説明
  → message: 追加例文でくらべる・パターンを見せる（任意）
  → message: 生徒が理解を確認
  → summary-point（要点1行）
  → quiz（理解度チェック1問）

date（次のテーマ）
  → narrator（次も例文から見ていこう、という誘導）
  → message: 先生が例文を提示（前のテーマとの対比も有効）
  → message: 生徒が変化・違いに気づく
  → message: 先生が説明
  → ...
  → summary-point
  → quiz

end（まとめ4-5点）
```

#### クイズ配置のルール（重要）

**1文法事項につき1問、説明の直後に配置する。**

- 否定文の説明をしたら → 否定文のクイズを1問
- 疑問文の説明をしたら → 疑問文のクイズを1問
- 短縮形の説明をしたら → 短縮形のクイズを1問
- 複数の文法事項をまとめて説明した後にクイズ1問、という構成にしない

**クイズで使う知識は、必ずその前の説明で教えておく。**

```
❌ 短縮形（isn't / aren't）を教える前に aren't を正解にしたクイズを出す
✅ 短縮形を先に教えてから、短縮形のクイズを出す
```

### 4.5 speakable（音声再生）ガイドライン

英語チャットでは、重要な例文に音声再生ボタンを表示できる。

#### speakableフィールド

MessageContent に `speakable?: string[]` を設定すると、吹き出しの下に音声再生ボタンが表示される。

```typescript
{
  type: 'message',
  side: 'left',
  characterId: 'teacher',
  expression: 'explaining',
  text: 'まずこの英文を読んでみよう！<br/><br/><strong>I am a student.</strong><br/>（私は生徒です。）',
  speakable: ['I am a student.'],  // ← 音声再生ボタンが表示される
}
```

#### 設定するメッセージの基準

- **先生メッセージの中で例文を初めて提示するとき** に設定する
- 生徒メッセージには設定しない
- 同じ英文が複数回出る場合は、最初に出るメッセージのみに設定

#### テキストの書き方

- 完全な英文を書く（`I am a student.` ピリオドや `?` も含む）
- HTMLタグは含めない（プレーンテキスト）
- 1メッセージあたり1〜3文が目安（多すぎるとボタンが並びすぎる）

## 5. 例題（examples）ガイドライン

### 5.1 ステップ設計の原則

**「何を考えればいいか分からない子」でも手順通りに進められる設計**

各ステップで明確な1アクション:
1. 日本語から「だれが？」を探す → 主語を決める
2. 主語に合う動詞の形を選ぶ
3. 残りの部分を英語にする

### 5.2 ステップの書き方

```typescript
{
  title: 'Step 1: 「だれが？」をさがそう',       // 短く、動詞で始める
  content: '「私は 生徒です」→「だれが？」→「私」だね。英語で I だよ。',  // やさしい語りかけ
  highlight: 'I',                                  // このステップの答え
}
```

- title: 「Step N: 〜しよう」の形
- content: 語りかけ口調で、1〜2文
- highlight: そのステップで得られるキーワード or 英語フレーズ

### 5.3 例題の構成パターン（中1）

1. **肯定文をつくる** — 日本語 → 英語の基本変換
2. **否定文をつくる** — 既存の英文を変換
3. **疑問文をつくる＋答え方** — 変換 + 応答

### 5.4 学年別の難易度調整

**中1**: 3ステップ。各ステップは1操作のみ。「だれが？」「どうする？」の日本語からのアプローチ。
**中2**: 3〜4ステップ。文の構造分析を少し含む。
**中3**: 3〜5ステップ。複数の文法事項を組み合わせる問題も可。

## 6. フラッシュカード（flashcards）ガイドライン

### 6.1 表示の仕組み（重要）

FlashcardDeck コンポーネントは **back を表面（最初に見える面）**、**front をフリップ後の裏面** として表示する。

したがって:
- **back**: 問題文（穴埋め形式の文。最初に表示される）
- **front**: 答え（1語〜短いフレーズ。フリップで表示される）
- hint: 考え方のヒント（中1は特に丁寧に）
- explanation: 正解の理由（1文）

### 6.2 データの書き方

```typescript
// FlashcardDeck では back→表面（先に見える）、front→裏面（フリップで見える）
{
  id: 'eng-be-fc1',
  front: 'am',                                    // ← 答え（フリップで見える）
  back: 'I ___ a student.（私は生徒です。）',      // ← 問題（最初に見える）
  hint: '「私 = I」のときのbe動詞は…？',
  explanation: 'I のときは、いつも am だよ。',
}
```

**注意**: front/back の名前と実際の表示が逆なので、必ずコメントで明示すること。歴史科目も同じ規則。

### 6.3 改行（`\n`）の使い方

テキスト内の `\n` は画面上で改行として表示される（`whitespace-pre-line` による）。**必要最小限の改行のみ使うこと。**

#### フラッシュカード back（穴埋め問題面）
- 英文と日本語訳の間に `\n` を1つ入れてよい（カード上の見やすさのため）
- 例: `'I (　　) a student.\n（私は生徒です。）'`

#### クイズ question
- **鉤括弧内の英文と日本語訳はインラインでまとめる**（`\n` を入れない）
- 鉤括弧の後と指示文の間に `\n` を1つだけ入れる
- OK: `'「I (　　) a student.（私は生徒です。）」\nの (　　) に入るのは？'`
- NG: `'「I (　　) a student.\n（私は生徒です。）」\nの (　　) に入るのは？'`

#### 例題 question
- 指示文は1行にまとめる（途中で `\n` を入れない）
- 対象の英文と日本語訳はインラインでまとめ、指示文との間に `\n` を1つだけ
- OK: `'次の文を「〜じゃない」の文にしよう。\nShe is a teacher.（彼女は先生です。）'`
- NG: `'次の文を「〜じゃない」の文にしよう。\nShe is a teacher.\n（彼女は先生です。）'`

#### 例題 answer
- 同義の英文表現は1行にまとめる
- 日本語訳との間に `\n` を1つだけ
- OK: `"She is not a teacher. = She isn't a teacher.\n（彼女は先生ではありません。）"`
- NG: `"She is not a teacher.\n= She isn't a teacher.\n（彼女は先生ではありません。）"`

## 7. クイズ（quiz）ガイドライン

- 4択。correctIndexの位置はバラけさせる
- explanation は文法用語を避け、理由をやさしく説明
- 中1: 基本的な形の選択問題が中心
- 中2-3: 文脈を含む問題、誤文訂正なども可

## 8. 品質チェックリスト

コンテンツ作成後、ファイルに書き込む前に確認:

### A. 言葉づかい
- [ ] 「補語」「文頭」「述語」「目的語」など禁止用語を使っていないか
- [ ] 学年に応じた文法用語のみ使用しているか
- [ ] 初出の文法用語にカッコ書き補足があるか（中1）
- [ ] 英文に日本語訳がついているか
- [ ] 難しい漢字にルビ（`<ruby>`タグ）が振られているか
- [ ] ひらがなのまま書かれた漢字語（いじょう、いちばん等）がないか

### B. 例題の質
- [ ] 各ステップが明確な1アクションか
- [ ] 「だれが？」「どうする？」など日本語からのアプローチがあるか（中1）
- [ ] highlight が各ステップに設定されているか
- [ ] 3パターン（肯定・否定・疑問）が含まれているか

### C. チャットの質（例文先行スタイル）
- [ ] 各シーンが例文の提示から始まっているか（ルール説明から入っていないか）
- [ ] 生徒が例文の中の変化や未知の部分に気づくリアクションをしているか
- [ ] 先生が例文をもとにかみ砕いて説明しているか
- [ ] 新しい文法事項の導入時に例文のくらべ技法を使っているか
- [ ] 先生の説明が1メッセージ2〜3文以内か
- [ ] 英文に日本語訳がセットになっているか
- [ ] summary-pointが各シーンにあるか

### D. 改行（`\n`）の適切さ
- [ ] クイズ question の鉤括弧内で英文と日本語訳の間に `\n` を入れていないか
- [ ] 例題 question の指示文が途中で `\n` で分断されていないか
- [ ] 例題 answer の同義英文が `\n` で分断されていないか
- [ ] 1テキストあたりの `\n` が最大1〜2個以内に収まっているか

### E. 技術的整合性
- [ ] IDが `eng-` プレフィックスで始まっているか
- [ ] chatId が chat.ts の id と一致しているか
- [ ] english-chat/index.ts に登録されているか
- [ ] TypeScriptコンパイルが通るか
